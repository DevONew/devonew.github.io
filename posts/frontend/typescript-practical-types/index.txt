JavaScript에서 TypeScript로 넘어오면서 가장 헷갈리는 부분이 바로 타입입니다. 특히 실무에서는 API 응답, 이벤트 핸들러, 복잡한 상태 관리 등 다양한 상황에서 타입을 정의해야 합니다. 

이 글에서는 실무에서 가장 자주 사용하는 타입 패턴들과, 많은 개발자들이 어려워하는 `any`를 안전하게 다루는 방법까지 총정리했습니다.

## 기본 타입 (Primitive Types)

TypeScript의 기본 타입부터 차근차근 살펴보겠습니다.

### 기본 타입 정의

```typescript
// 문자열
const name: string = 'Alice';
const greeting: string = `Hello, ${name}`;

// 숫자
const age: number = 25;
const price: number = 19.99;
const binary: number = 0b1010; // 2진수
const hex: number = 0xf00d; // 16진수

// 불리언
const isActive: boolean = true;
const hasPermission: boolean = false;

// null과 undefined
const nothing: null = null;
const notDefined: undefined = undefined;

// symbol (ES6)
const sym: symbol = Symbol('unique');

// bigint (ES2020)
const bigNumber: bigint = 100n;
```

**실무 팁**: `strictNullChecks` 옵션을 켜면 `null`과 `undefined`를 명시적으로 처리해야 합니다.

```typescript
// strictNullChecks: true일 때
let name: string = 'Alice';
name = null; // ❌ 에러!

let nullableName: string | null = 'Bob';
nullableName = null; // ✅ OK
```

### 배열 타입

```typescript
// 방법 1: 타입[]
const numbers: number[] = [1, 2, 3, 4, 5];
const names: string[] = ['Alice', 'Bob', 'Charlie'];

// 방법 2: Array<타입>
const scores: Array<number> = [90, 85, 95];
const emails: Array<string> = ['a@test.com', 'b@test.com'];

// 다차원 배열
const matrix: number[][] = [
  [1, 2, 3],
  [4, 5, 6],
  [7, 8, 9]
];

// 읽기 전용 배열
const readonlyArr: readonly number[] = [1, 2, 3];
// readonlyArr.push(4); // ❌ 에러!

// ReadonlyArray 타입
const readonlyArr2: ReadonlyArray<string> = ['a', 'b', 'c'];
```

**실무 활용**: React에서 상태를 불변으로 관리할 때 readonly 사용

```typescript
interface State {
  readonly items: readonly string[];
}

// 새 배열을 반환해야 함
function addItem(state: State, item: string): State {
  return {
    items: [...state.items, item]
  };
}
```

### 튜플 (Tuple)

고정된 개수와 타입의 배열을 표현합니다.

```typescript
// 기본 튜플
const user: [string, number] = ['Alice', 25];
const user2: [string, number] = [25, 'Alice']; // ❌ 타입 순서 불일치

// 튜플 요소 접근
const name = user[0]; // string
const age = user[1];  // number

// 라벨이 있는 튜플 (TS 4.0+)
type User = [name: string, age: number, isAdmin: boolean];
const admin: User = ['Alice', 25, true];

// 선택적 요소
type Response = [status: number, data: string, error?: string];
const success: Response = [200, 'OK'];
const failure: Response = [500, 'Server Error', 'Internal error'];

// Rest 요소
type StringNumberBooleans = [string, number, ...boolean[]];
const example: StringNumberBooleans = ['hello', 1, true, false, true];

// 실무 예시: React useState
const [count, setCount] = useState(0); // [number, Dispatch<SetStateAction<number>>]
```

**실무 활용**: 함수에서 여러 값을 반환할 때

```typescript
function getUser(id: number): [User | null, Error | null] {
  try {
    const user = fetchUser(id);
    return [user, null];
  } catch (error) {
    return [null, error as Error];
  }
}

const [user, error] = getUser(1);
if (error) {
  console.error(error);
} else {
  console.log(user?.name);
}
```

## 객체 타입

### Interface vs Type Alias

```typescript
// Interface
interface User {
  id: number;
  name: string;
  email: string;
}

// Type Alias
type Product = {
  id: number;
  name: string;
  price: number;
};
```

**언제 무엇을 사용할까?**

**Interface를 사용하는 경우:**
- 객체의 형태를 정의할 때
- 확장(extends)이 필요할 때
- 선언 병합(declaration merging)이 필요할 때

```typescript
interface Animal {
  name: string;
}

// 확장
interface Dog extends Animal {
  bark(): void;
}

// 선언 병합
interface User {
  id: number;
}

interface User {
  name: string;
}

// 자동으로 병합됨
const user: User = {
  id: 1,
  name: 'Alice'
};
```

**Type Alias를 사용하는 경우:**
- Union, Intersection 타입
- Tuple, Primitive 타입에 이름 붙일 때
- 복잡한 타입 조합

```typescript
// Union
type Status = 'pending' | 'approved' | 'rejected';

// Intersection
type Admin = User & {
  role: 'admin';
  permissions: string[];
};

// Tuple
type Point = [number, number];

// Mapped type
type Readonly<T> = {
  readonly [P in keyof T]: T[P];
};
```

### 선택적 프로퍼티와 읽기 전용

```typescript
interface User {
  id: number;
  name: string;
  email?: string; // 선택적
  readonly createdAt: Date; // 읽기 전용
  age?: number;
}

const user: User = {
  id: 1,
  name: 'Alice',
  createdAt: new Date()
};

// user.createdAt = new Date(); // ❌ 읽기 전용 속성 변경 불가
user.email = 'alice@example.com'; // ✅ 선택적 속성 추가 가능
```

### 인덱스 시그니처

동적 프로퍼티를 가진 객체의 타입을 정의합니다.

```typescript
// 문자열 인덱스
interface StringMap {
  [key: string]: string;
}

const translations: StringMap = {
  hello: '안녕하세요',
  goodbye: '안녕히 가세요',
  thanks: '감사합니다'
};

// 숫자 인덱스
interface NumberMap {
  [index: number]: string;
}

const items: NumberMap = {
  0: 'first',
  1: 'second',
  2: 'third'
};

// 혼합 (특정 키 + 인덱스)
interface Config {
  port: number; // 필수 속성
  [key: string]: number | string; // 추가 속성
}

const config: Config = {
  port: 3000,
  host: 'localhost',
  timeout: 5000
};

// Record 유틸리티 타입으로 대체 가능
type ErrorMessages = Record<string, string>;
const errors: ErrorMessages = {
  required: '필수 항목입니다',
  email: '이메일 형식이 올바르지 않습니다'
};
```

## 함수 타입

### 함수 선언과 타입

```typescript
// 기본 함수 타입
function add(a: number, b: number): number {
  return a + b;
}

// 화살표 함수
const multiply = (a: number, b: number): number => {
  return a * b;
};

// 함수 타입 정의
type MathOperation = (a: number, b: number) => number;

const subtract: MathOperation = (a, b) => a - b;
const divide: MathOperation = (a, b) => a / b;

// 선택적 매개변수
function greet(name: string, greeting?: string): string {
  return `${greeting || 'Hello'}, ${name}!`;
}

greet('Alice'); // "Hello, Alice!"
greet('Bob', 'Hi'); // "Hi, Bob!"

// 기본값 매개변수
function createUser(name: string, age: number = 18): User {
  return { id: Date.now(), name, age };
}

// Rest 매개변수
function sum(...numbers: number[]): number {
  return numbers.reduce((total, n) => total + n, 0);
}

sum(1, 2, 3, 4, 5); // 15
```

### 함수 오버로딩

같은 이름의 함수에 여러 시그니처를 제공합니다.

```typescript
// 오버로드 시그니처
function format(value: string): string;
function format(value: number): string;
function format(value: Date): string;

// 구현 시그니처
function format(value: string | number | Date): string {
  if (typeof value === 'string') {
    return value.toUpperCase();
  }
  if (typeof value === 'number') {
    return value.toFixed(2);
  }
  return value.toISOString();
}

format('hello'); // "HELLO"
format(42); // "42.00"
format(new Date()); // "2024-10-08T..."

// 실무 예시: API 함수
function fetchData(id: number): Promise<User>;
function fetchData(ids: number[]): Promise<User[]>;

async function fetchData(input: number | number[]): Promise<User | User[]> {
  if (Array.isArray(input)) {
    return Promise.all(input.map(id => fetchUser(id)));
  }
  return fetchUser(input);
}
```

### void, never, unknown 반환 타입

```typescript
// void - 반환값 없음
function log(message: string): void {
  console.log(message);
  // return 문이 없거나 return;만 가능
}

// never - 절대 반환하지 않음
function throwError(message: string): never {
  throw new Error(message);
}

function infiniteLoop(): never {
  while (true) {
    // 무한 루프
  }
}

// never는 모든 타입의 서브타입 (유니온에서 제거됨)
type T1 = string | never; // string
type T2 = number | never; // number

// unknown - 타입 안전한 any
function parseJSON(json: string): unknown {
  return JSON.parse(json);
}

const result = parseJSON('{"name": "Alice"}');
// result.name; // ❌ 에러! unknown 타입은 바로 사용 불가

// 타입 체크 후 사용
if (typeof result === 'object' && result !== null && 'name' in result) {
  console.log((result as { name: string }).name);
}
```

## Union과 Intersection 타입

### Union Types (|)

여러 타입 중 하나를 가질 수 있습니다.

```typescript
// 기본 Union
type ID = string | number;

let userId: ID = 123;
userId = 'abc-123';

// 리터럴 타입 Union
type Status = 'idle' | 'loading' | 'success' | 'error';

function handleStatus(status: Status) {
  switch (status) {
    case 'idle':
      return '대기 중';
    case 'loading':
      return '로딩 중...';
    case 'success':
      return '완료!';
    case 'error':
      return '오류 발생';
  }
}

// 객체 Union
type Circle = {
  kind: 'circle';
  radius: number;
};

type Rectangle = {
  kind: 'rectangle';
  width: number;
  height: number;
};

type Shape = Circle | Rectangle;

function getArea(shape: Shape): number {
  switch (shape.kind) {
    case 'circle':
      return Math.PI * shape.radius ** 2;
    case 'rectangle':
      return shape.width * shape.height;
  }
}

// Discriminated Union (태그된 유니온)
type Success<T> = {
  type: 'success';
  data: T;
};

type Failure = {
  type: 'error';
  error: string;
};

type Result<T> = Success<T> | Failure;

function handleResult<T>(result: Result<T>) {
  if (result.type === 'success') {
    console.log(result.data); // T 타입으로 좁혀짐
  } else {
    console.error(result.error); // string 타입
  }
}
```

### Intersection Types (&)

여러 타입을 모두 만족해야 합니다.

```typescript
// 기본 Intersection
type Person = {
  name: string;
  age: number;
};

type Employee = {
  employeeId: number;
  department: string;
};

type Worker = Person & Employee;

const worker: Worker = {
  name: 'Alice',
  age: 30,
  employeeId: 12345,
  department: 'Engineering'
};

// Mixin 패턴
interface Timestamped {
  createdAt: Date;
  updatedAt: Date;
}

interface Deletable {
  deletedAt?: Date;
}

type Post = {
  id: number;
  title: string;
  content: string;
} & Timestamped & Deletable;

const post: Post = {
  id: 1,
  title: 'Hello',
  content: 'World',
  createdAt: new Date(),
  updatedAt: new Date()
};

// 함수 타입 Intersection
type Logger = {
  log: (message: string) => void;
};

type Formatter = {
  format: (data: unknown) => string;
};

type LoggerWithFormatter = Logger & Formatter;

const logger: LoggerWithFormatter = {
  log: (message) => console.log(message),
  format: (data) => JSON.stringify(data)
};
```

## Type Narrowing (타입 좁히기)

TypeScript가 타입을 더 구체적으로 추론하도록 돕는 기법입니다.

### typeof 타입 가드

```typescript
function processValue(value: string | number) {
  if (typeof value === 'string') {
    // 이 블록 안에서 value는 string 타입
    return value.toUpperCase();
  }
  // 이 블록 안에서 value는 number 타입
  return value.toFixed(2);
}

// 실무 예시: 다양한 타입 처리
function formatInput(input: string | number | boolean): string {
  if (typeof input === 'string') {
    return input.trim();
  }
  if (typeof input === 'number') {
    return input.toString();
  }
  return input ? 'Yes' : 'No';
}
```

### instanceof 타입 가드

```typescript
class Dog {
  bark() {
    console.log('Woof!');
  }
}

class Cat {
  meow() {
    console.log('Meow!');
  }
```

```typescript
  delay: number = 1000
```

## any 완벽 정복

`any`는 TypeScript의 타입 시스템을 완전히 우회하는 escape hatch입니다. 편리하지만 위험하므로, 올바르게 사용하는 방법을 알아야 합니다.

### any가 위험한 이유

```typescript
// ❌ any는 모든 타입 체크를 무력화
let data: any = 'hello';

data.toUpperCase(); // OK
data.toFixed(); // OK... 런타임에서 에러!
data.nonExistentMethod(); // OK... 런타임에서 에러!

// 컴파일 타임에 잡히지 않는 버그
function processUser(user: any) {
  console.log(user.name.toUpperCase()); // user.name이 없으면?
  console.log(user.age + 1); // age가 문자열이면?
}

processUser({}); // 런타임 에러!
processUser({ name: 123 }); // 런타임 에러!

// 타입 추론 망가뜨림
const result: any = fetchData();
const value = result.data; // value는 any 타입
const doubled = value * 2; // doubled도 any 타입
```

```typescript
  // 기존 JavaScript 코드
  return { id: 1, name: 'Test' };
}

let data: any = fetchLegacyData();

// Step 2: 사용 패턴 파악 및 타입 정의
interface LegacyData {
  id: number;
  name: string;
  // 점진적으로 필드 추가
}

// Step 3: 타입 가드 작성
function isLegacyData(data: unknown): data is LegacyData {
  return (
    typeof data === 'object' &&
    data !== null &&
    'id' in data &&
    typeof (data as LegacyData).id === 'number' &&
    'name' in data &&
    typeof (data as LegacyData).name === 'string'
  );
}

// Step 4: 점진적으로 타입 적용
function fetchLegacyData(): unknown {
  return { id: 1, name: 'Test' };
}

const data = fetchLegacyData();
if (isLegacyData(data)) {
  console.log(data.name); // 타입 안전
}

// Step 5: 완전한 타입 시스템으로 전환
function fetchLegacyData(): LegacyData {
  const data = { id: 1, name: 'Test' };
  // 내부적으로 검증 로직 추가
  if (!isLegacyData(data)) {
    throw new Error('Invalid data format');
  }
  return data;
}
```

```typescript
// Redux
interface AppState {
  user: User | null;
  isLoading: boolean;
  error: string | null;
  posts: Post[];
  comments: Record<number, Comment[]>;
}

// Action Types
type Action =
  | { type: 'USER_LOGIN'; payload: User }
  | { type: 'USER_LOGOUT' }
  | { type: 'SET_LOADING'; payload: boolean }
  | { type: 'SET_ERROR'; payload: string }
  | { type: 'ADD_POST'; payload: Post }
  | { type: 'UPDATE_POST'; payload: { id: number; updates: Partial<Post> } }
  | { type: 'DELETE_POST'; payload: number }
  | { type: 'ADD_COMMENT'; payload: { postId: number; comment: Comment } };

// Reducer
function reducer(state: AppState, action: Action): AppState {
  switch (action.type) {
    case 'USER_LOGIN':
      return { ...state, user: action.payload, error: null };

    case 'USER_LOGOUT':
      return { ...state, user: null };

    case 'SET_LOADING':
      return { ...state, isLoading: action.payload };

    case 'SET_ERROR':
      return { ...state, error: action.payload };

    case 'ADD_POST':
      return { ...state, posts: [...state.posts, action.payload] };

    case 'UPDATE_POST':
      return {
        ...state,
        posts: state.posts.map(post =>
          post.id === action.payload.id
            ? { ...post, ...action.payload.updates }
            : post
        )
      };

    case 'DELETE_POST':
      return {
        ...state,
        posts: state.posts.filter(post => post.id !== action.payload)
      };

    case 'ADD_COMMENT':
      const { postId, comment } = action.payload;
      return {
        ...state,
        comments: {
          ...state.comments,
          [postId]: [...(state.comments[postId] || []), comment]
        }
      };

    default:
      return state;
  }
}
```

## 마치며

TypeScript의 타입 시스템은 강력하지만, 올바르게 사용하는 것이 중요합니다.

**핵심 원칙:**
1. **any 최소화**: unknown, 제네릭, 유틸리티 타입으로 대체
2. **타입 좁히기**: typeof, in, instanceof, 사용자 정의 타입 가드 활용
3. **재사용성**: 제네릭과 유틸리티 타입으로 DRY 원칙 지키기
4. **점진적 적용**: 레거시 코드는 단계적으로 마이그레이션
5. **명확한 의도**: 타입이 복잡할 땐 주석으로 설명

처음에는 타입 정의가 번거롭게 느껴질 수 있지만, 한번 제대로 정의해두면:
- 자동완성으로 개발 속도 향상
- 리팩토링이 안전하고 쉬워짐
- 런타임 에러가 컴파일 타임에 잡힘
- 코드가 문서화되어 협업이 쉬워짐
