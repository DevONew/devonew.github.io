TypeScript 실무 타입 정리 + any 지양하기

📅2025년 10월 8일
프론트엔드TypeScript타입타입안전성

JavaScript에서 TypeScript로 넘어오면서 가장 헷갈리는 부분이 바로 타입입니다. 특히 실무에서는 API 응답, 이벤트 핸들러, 복잡한 상태 관리 등 다양한 상황에서 타입을 정의해야 합니다.

이 글에서는 실무에서 가장 자주 사용하는 타입 패턴들과, 많은 개발자들이 어려워하는 any를 안전하게 다루는 방법까지 총정리했습니다.

기본 타입 (Primitive Types)

TypeScript의 기본 타입부터 차근차근 살펴보겠습니다.

기본 타입 정의

// 문자열
const name: string = 'Alice';
const greeting: string = `Hello, ${name}`;

// 숫자
const age: number = 25;
const price: number = 19.99;
const binary: number = 0b1010; // 2진수
const hex: number = 0xf00d; // 16진수

// 불리언
const isActive: boolean = true;
const hasPermission: boolean = false;

// null과 undefined
const nothing: null = null;
const notDefined: undefined = undefined;

// symbol (ES6)
const sym: symbol = Symbol('unique');

// bigint (ES2020)
const bigNumber: bigint = 100n;

실무 팁: strictNullChecks 옵션을 켜면 null과 undefined를 명시적으로 처리해야 합니다.

// strictNullChecks: true일 때
let name: string = 'Alice';
name = null; // ❌ 에러!

let nullableName: string | null = 'Bob';
nullableName = null; // ✅ OK

// 방법 2: Array<타입>

  [7, 8, 9]

실무 활용: React에서 상태를 불변으로 관리할 때 readonly 사용

interface State {
  readonly items: readonly string[];
}

// 새 배열을 반환해야 함
function addItem(state: State, item: string): State {
  return {
    items: [...state.items, item]
  };
}

튜플 (Tuple)

고정된 개수와 타입의 배열을 표현합니다.

// 기본 튜플
const user: [string, number] = ['Alice', 25];
const user2: [string, number] = [25, 'Alice']; // ❌ 타입 순서 불일치

// 튜플 요소 접근
const name = user[0]; // string
const age = user[1];  // number

// 라벨이 있는 튜플 (TS 4.0+)
type User = [name: string, age: number, isAdmin: boolean];
const admin: User = ['Alice', 25, true];

// 선택적 요소
type Response = [status: number, data: string, error?: string];
const success: Response = [200, 'OK'];
const failure: Response = [500, 'Server Error', 'Internal error'];

// Rest 요소
type StringNumberBooleans = [string, number, ...boolean[]];
const example: StringNumberBooleans = ['hello', 1, true, false, true];

// 실무 예시: React useState
const [count, setCount] = useState(0); // [number, Dispatch<SetStateAction<number>>]

실무 활용: 함수에서 여러 값을 반환할 때

function getUser(id: number): [User | null, Error | null] {
  try {
    const user = fetchUser(id);
    return [user, null];
  } catch (error) {
    return [null, error as Error];
  }
}

const [user, error] = getUser(1);
if (error) {
  console.error(error);
} else {
  console.log(user?.name);
}

객체 타입

Interface vs Type Alias

// Interface
interface User {
  id: number;
  name: string;
  email: string;
}

// Type Alias
type Product = {
  id: number;
  name: string;
  price: number;
};

언제 무엇을 사용할까?

Interface를 사용하는 경우:

  • 객체의 형태를 정의할 때
  • 확장(extends)이 필요할 때
  • 선언 병합(declaration merging)이 필요할 때
interface Animal {
  name: string;
}

// 확장
interface Dog extends Animal {
  bark(): void;
}

// 선언 병합
interface User {
  id: number;
}

interface User {
  name: string;
}

// 자동으로 병합됨
const user: User = {
  id: 1,
  name: 'Alice'
};

Type Alias를 사용하는 경우:

  • Union, Intersection 타입
  • Tuple, Primitive 타입에 이름 붙일 때
  • 복잡한 타입 조합
// Union
type Status = 'pending' | 'approved' | 'rejected';

// Intersection
type Admin = User & {
  role: 'admin';
  permissions: string[];
};

// Tuple
type Point = [number, number];

// Mapped type
type Readonly<T> = {
  readonly [P in keyof T]: T[P];
};

함수 타입

함수 선언과 타입

// 기본 함수 타입
function add(a: number, b: number): number {
  return a + b;
}

// 화살표 함수
const multiply = (a: number, b: number): number => {
  return a * b;
};

// 함수 타입 정의
type MathOperation = (a: number, b: number) => number;

const subtract: MathOperation = (a, b) => a - b;
const divide: MathOperation = (a, b) => a / b;

// 선택적 매개변수
function greet(name: string, greeting?: string): string {
  return `${greeting || 'Hello'}, ${name}!`;
}

greet('Alice'); // "Hello, Alice!"
greet('Bob', 'Hi'); // "Hi, Bob!"

// 기본값 매개변수
function createUser(name: string, age: number = 18): User {
  return { id: Date.now(), name, age };
}

// Rest 매개변수
function sum(...numbers: number[]): number {
  return numbers.reduce((total, n) => total + n, 0);
}

sum(1, 2, 3, 4, 5); // 15

함수 오버로딩

같은 이름의 함수에 여러 시그니처를 제공합니다.

// 오버로드 시그니처
function format(value: string): string;
function format(value: number): string;
function format(value: Date): string;

// 구현 시그니처
function format(value: string | number | Date): string {
  if (typeof value === 'string') {
    return value.toUpperCase();
  }
  if (typeof value === 'number') {
    return value.toFixed(2);
  }
  return value.toISOString();
}

format('hello'); // "HELLO"
format(42); // "42.00"
format(new Date()); // "2024-10-08T..."

// 실무 예시: API 함수
function fetchData(id: number): Promise<User>;
function fetchData(ids: number[]): Promise<User[]>;

async function fetchData(input: number | number[]): Promise<User | User[]> {
  if (Array.isArray(input)) {
    return Promise.all(input.map(id => fetchUser(id)));
  }
  return fetchUser(input);
}

void, never, unknown 반환 타입

// void - 반환값 없음
function log(message: string): void {
  console.log(message);
  // return 문이 없거나 return;만 가능
}

// never - 절대 반환하지 않음
function throwError(message: string): never {
  throw new Error(message);
}

function infiniteLoop(): never {
  while (true) {
    // 무한 루프
  }
}

// never는 모든 타입의 서브타입 (유니온에서 제거됨)
type T1 = string | never; // string
type T2 = number | never; // number

// unknown - 타입 안전한 any
function parseJSON(json: string): unknown {
  return JSON.parse(json);
}

const result = parseJSON('{"name": "Alice"}');
// result.name; // ❌ 에러! unknown 타입은 바로 사용 불가

// 타입 체크 후 사용
if (typeof result === 'object' && result !== null && 'name' in result) {
  console.log((result as { name: string }).name);
}

Union과 Intersection 타입

Union Types (|)

여러 타입 중 하나를 가질 수 있습니다.

// 기본 Union
type ID = string | number;

let userId: ID = 123;
userId = 'abc-123';

// 리터럴 타입 Union
type Status = 'idle' | 'loading' | 'success' | 'error';

function handleStatus(status: Status) {
  switch (status) {
    case 'idle':
      return '대기 중';
    case 'loading':
      return '로딩 중...';
    case 'success':
      return '완료!';
    case 'error':
      return '오류 발생';
  }
}

// 객체 Union
type Circle = {
  kind: 'circle';
  radius: number;
};

type Rectangle = {
  kind: 'rectangle';
  width: number;
  height: number;
};

type Shape = Circle | Rectangle;

function getArea(shape: Shape): number {
  switch (shape.kind) {
    case 'circle':
      return Math.PI * shape.radius ** 2;
    case 'rectangle':
      return shape.width * shape.height;
  }
}

// Discriminated Union (태그된 유니온)
type Success<T> = {
  type: 'success';
  data: T;
};

type Failure = {
  type: 'error';
  error: string;
};

type Result<T> = Success<T> | Failure;

function handleResult<T>(result: Result<T>) {
  if (result.type === 'success') {
    console.log(result.data); // T 타입으로 좁혀짐
  } else {
    console.error(result.error); // string 타입
  }
}

Intersection Types (&)

여러 타입을 모두 만족해야 합니다.

// 기본 Intersection
type Person = {
  name: string;
  age: number;
};

type Employee = {
  employeeId: number;
  department: string;
};

type Worker = Person & Employee;

const worker: Worker = {
  name: 'Alice',
  age: 30,
  employeeId: 12345,
  department: 'Engineering'
};

// Mixin 패턴
interface Timestamped {
  createdAt: Date;
  updatedAt: Date;
}

interface Deletable {
  deletedAt?: Date;
}

type Post = {
  id: number;
  title: string;
  content: string;
} & Timestamped & Deletable;

const post: Post = {
  id: 1,
  title: 'Hello',
  content: 'World',
  createdAt: new Date(),
  updatedAt: new Date()
};

// 함수 타입 Intersection
type Logger = {
  log: (message: string) => void;
};

type Formatter = {
  format: (data: unknown) => string;
};

type LoggerWithFormatter = Logger & Formatter;

const logger: LoggerWithFormatter = {
  log: (message) => console.log(message),
  format: (data) => JSON.stringify(data)
};

Type Narrowing (타입 좁히기)

TypeScript가 타입을 더 구체적으로 추론하도록 돕는 기법입니다.

typeof 타입 가드

function processValue(value: string | number) {
  if (typeof value === 'string') {
    // 이 블록 안에서 value는 string 타입
    return value.toUpperCase();
  }
  // 이 블록 안에서 value는 number 타입
  return value.toFixed(2);
}

// 실무 예시: 다양한 타입 처리
function formatInput(input: string | number | boolean): string {
  if (typeof input === 'string') {
    return input.trim();
  }
  if (typeof input === 'number') {
    return input.toString();
  }
  return input ? 'Yes' : 'No';
}

instanceof 타입 가드

class Dog {
  bark() {
    console.log('Woof!');
  }
}

class Cat {
  meow() {
    console.log('Meow!');
  }
}

function makeSound(animal: Dog | Cat) {
  if (animal instanceof Dog) {
    animal.bark();
  } else {
    animal.meow();
  }
}

// 에러 처리
function handleError(error: Error | string) {
  if (error instanceof Error) {
    console.error(error.message);
    console.error(error.stack);
  } else {
    console.error(error);
  }
}

// 날짜 처리
function formatDate(date: Date | string): string {
  if (date instanceof Date) {
    return date.toISOString();
  }
  return new Date(date).toISOString();
}

사용자 정의 타입 가드

// 기본 타입 가드
function isString(value: unknown): value is string {
  return typeof value === 'string';
}

function isNumber(value: unknown): value is number {
  return typeof value === 'number';
}

// 객체 타입 가드
interface User {
  id: number;
  name: string;
  email: string;
}

function isUser(obj: unknown): obj is User {
  return (
    typeof obj === 'object' &&
    obj !== null &&
    'id' in obj &&
    typeof (obj as User).id === 'number' &&
    'name' in obj &&
    typeof (obj as User).name === 'string' &&
    'email' in obj &&
    typeof (obj as User).email === 'string'
  );
}

// 사용
const data: unknown = fetchData();

if (isUser(data)) {
  console.log(data.name); // ✅ User 타입으로 안전하게 사용
}

// 배열 타입 가드
function isStringArray(value: unknown): value is string[] {
  return (
    Array.isArray(value) &&
    value.every(item => typeof item === 'string')
  );
}

// 제네릭 타입 가드
function isArrayOf<T>(
  arr: unknown,
  check: (item: unknown) => item is T
): arr is T[] {
  return Array.isArray(arr) && arr.every(check);
}

const data: unknown = ['a', 'b', 'c'];
if (isArrayOf(data, isString)) {
  data.forEach(str => console.log(str.toUpperCase()));
}

Discriminated Union 패턴

// API 응답 타입
type ApiResponse<T> =
  | { status: 'loading' }
  | { status: 'success'; data: T }
  | { status: 'error'; error: string };

function handleResponse<T>(response: ApiResponse<T>) {
  switch (response.status) {
    case 'loading':
      console.log('Loading...');
      break;
    case 'success':
      console.log(response.data); // T 타입
      break;
    case 'error':
      console.error(response.error); // string 타입
      break;
  }
}

// 실무 예시: 폼 필드 타입
type TextField = {
  type: 'text';
  value: string;
  placeholder?: string;
};

type NumberField = {
  type: 'number';
  value: number;
  min?: number;
  max?: number;
};

type SelectField = {
  type: 'select';
  value: string;
  options: string[];
};

type FormField = TextField | NumberField | SelectField;

function renderField(field: FormField) {
  switch (field.type) {
    case 'text':
      return `<input type="text" value="${field.value}" placeholder="${field.placeholder || ''}" />`;
    case 'number':
      return `<input type="number" value="${field.value}" min="${field.min}" max="${field.max}" />`;
    case 'select':
      return `<select>${field.options.map(opt => `<option>${opt}</option>`).join('')}</select>`;
  }
}

제네릭 (Generics)

재사용 가능한 타입을 만드는 강력한 도구입니다.

기본 제네릭

// 기본 제네릭 함수
function identity<T>(value: T): T {
  return value;
}

const num = identity<number>(42); // number
const str = identity<string>('hello'); // string
const auto = identity(true); // boolean (타입 추론)

// 배열 제네릭
function getFirstElement<T>(arr: T[]): T | undefined {
  return arr[0];
}

const first = getFirstElement([1, 2, 3]); // number | undefined
const firstStr = getFirstElement(['a', 'b']); // string | undefined

// 여러 제네릭 파라미터
function pair<T, U>(first: T, second: U): [T, U] {
  return [first, second];
}

const p1 = pair(1, 'hello'); // [number, string]
const p2 = pair('id', { name: 'Alice' }); // [string, { name: string }]

제네릭 인터페이스와 타입

// 제네릭 인터페이스
interface Box<T> {
  value: T;
}

const numBox: Box<number> = { value: 42 };
const strBox: Box<string> = { value: 'hello' };

// 제네릭 타입 별칭
type Result<T, E = Error> = 
  | { success: true; value: T }
  | { success: false; error: E };

const success: Result<User> = {
  success: true,
  value: { id: 1, name: 'Alice', email: 'alice@example.com' }
};

const failure: Result<User, string> = {
  success: false,
  error: 'User not found'
};

// 제네릭 클래스
class Stack<T> {
  private items: T[] = [];

  push(item: T): void {
    this.items.push(item);
  }

  pop(): T | undefined {
    return this.items.pop();
  }

  peek(): T | undefined {
    return this.items[this.items.length - 1];
  }

  get size(): number {
    return this.items.length;
  }
}

const numberStack = new Stack<number>();
numberStack.push(1);
numberStack.push(2);
console.log(numberStack.pop()); // 2

제네릭 제약 조건

// extends로 제약
interface HasId {
  id: number;
}

function findById<T extends HasId>(items: T[], id: number): T | undefined {
  return items.find(item => item.id === id);
}

const users = [
  { id: 1, name: 'Alice' },
  { id: 2, name: 'Bob' }
];

const user = findById(users, 1); // { id: number; name: string } | undefined

// findById(['a', 'b'], 1); // ❌ 에러! string은 HasId를 만족하지 않음

// keyof 제약
function getProperty<T, K extends keyof T>(obj: T, key: K): T[K] {
  return obj[key];
}

const user = { id: 1, name: 'Alice', age: 25 };
const name = getProperty(user, 'name'); // string
const age = getProperty(user, 'age'); // number
// getProperty(user, 'invalid'); // ❌ 에러!

// 여러 제약 조건
interface Lengthwise {
  length: number;
}

function logLength<T extends Lengthwise>(item: T): void {
  console.log(item.length);
}

logLength('hello'); // ✅ string은 length 속성 있음
logLength([1, 2, 3]); // ✅ 배열도 length 속성 있음
logLength({ length: 10, value: 'test' }); // ✅ 명시적으로 length 있음
// logLength(42); // ❌ number는 length 속성 없음

실무 제네릭 패턴

// API 함수
async function fetchData<T>(url: string): Promise<T> {
  const response = await fetch(url);
  if (!response.ok) {
    throw new Error(`HTTP error! status: ${response.status}`);
  }
  return response.json();
}

interface User {
  id: number;
  name: string;
  email: string;
}

const user = await fetchData<User>('/api/user/1');

// Promise 래퍼
async function retry<T>(
  fn: () => Promise<T>,
  maxAttempts: number = 3,
  delay: number = 1000
): Promise<T> {
  for (let attempt = 1; attempt <= maxAttempts; attempt++) {
    try {
      return await fn();
    } catch (error) {
      if (attempt === maxAttempts) throw error;
      await new Promise(resolve => setTimeout(resolve, delay));
    }
  }
  throw new Error('Max attempts reached');
}

interface ListProps<T> {
  items: T[];
  renderItem: (item: T, index: number) => React.ReactNode;
  keyExtractor: (item: T) => string | number;
  emptyMessage?: string;
}

function List<T>({
  items,
  renderItem,
  keyExtractor,
  emptyMessage = 'No items'
}: ListProps<T>) {
  if (items.length === 0) {
    return <div>{emptyMessage}</div>;
  }

  return (
    <ul>
      {items.map((item, index) => (
        <li key={keyExtractor(item)}>
          {renderItem(item, index)}
        </li>
      ))}
    </ul>
  );
}

    case 'USER_LOGIN':
      return { ...state, user: action.payload, error: null };

    case 'USER_LOGOUT':
      return { ...state, user: null };

    case 'SET_LOADING':
      return { ...state, isLoading: action.payload };

    case 'SET_ERROR':
      return { ...state, error: action.payload };

    case 'ADD_POST':
      return { ...state, posts: [...state.posts, action.payload] };

    case 'UPDATE_POST':
      return {
        ...state,
        posts: state.posts.map(post =>
          post.id === action.payload.id
            ? { ...post, ...action.payload.updates }
            : post
        )
      };

    case 'DELETE_POST':
      return {
        ...state,
        posts: state.posts.filter(post => post.id !== action.payload)
      };

    case 'ADD_COMMENT':
      const { postId, comment } = action.payload;
      return {
        ...state,
        comments: {
          ...state.comments,
          [postId]: [...(state.comments[postId] || []), comment]
        }
      };

    default:
      return state;
  }
}

// Zustand
interface StoreState {
  count: number;
  user: User | null;
  increment: () => void;
  decrement: () => void;
  setUser: (user: User | null) => void;
  reset: () => void;
}

const useStore = create<StoreState>((set) => ({
  count: 0,
  user: null,
  increment: () => set((state) => ({ count: state.count + 1 })),
  decrement: () => set((state) => ({ count: state.count - 1 })),
  setUser: (user) => set({ user }),
  reset: () => set({ count: 0, user: null })
}));

Mapped Types

// 모든 속성을 선택적으로
type Optional<T> = {
  [P in keyof T]?: T[P];
};

// 모든 속성을 nullable로
type Nullable<T> = {
  [P in keyof T]: T[P] | null;
};

// 특정 키만 선택적으로
type PartialBy<T, K extends keyof T> = Omit<T, K> & Partial<Pick<T, K>>;

interface User {
  id: number;
  name: string;
  email: string;
  age: number;
}

// name과 email만 선택적
type UserWithOptionalContact = PartialBy<User, 'name' | 'email'>;

// 함수 타입을 Promise로 변환
type Promisify<T> = {
  [P in keyof T]: T[P] extends (...args: infer A) => infer R
    ? (...args: A) => Promise<R>
    : T[P];
};

interface API {
  getUser: (id: number) => User;
  deleteUser: (id: number) => void;
}

type AsyncAPI = Promisify<API>;
// {
//   getUser: (id: number) => Promise<User>;
//   deleteUser: (id: number) => Promise<void>;

Conditional Types

// 기본 조건부 타입
type IsString<T> = T extends string ? true : false;

type A = IsString<string>; // true
type B = IsString<number>; // false

// 실용적 예시
type NonNullable<T> = T extends null | undefined ? never : T;

type T1 = NonNullable<string | null>; // string
type T2 = NonNullable<number | undefined>; // number

// 중첩 조건부 타입
type TypeName<T> = 
  T extends string ? 'string' :
  T extends number ? 'number' :
  T extends boolean ? 'boolean' :
  T extends undefined ? 'undefined' :
  T extends Function ? 'function' :
  'object';

type T3 = TypeName<string>; // 'string'
type T4 = TypeName<() => void>; // 'function'

// Flatten 타입 (배열을 펼침)
type Flatten<T> = T extends Array<infer U> ? U : T;

type T5 = Flatten<string[]>; // string
type T6 = Flatten<number>; // number

Template Literal Types

// 기본 사용
type World = 'world';
type Greeting = `hello ${World}`; // 'hello world'

// 이벤트 핸들러 타입
type EventName = 'click' | 'scroll' | 'mousemove';
type Handler = `on${Capitalize<EventName>}`;

// CSS 속성
type CSSProperty = 'color' | 'font-size' | 'background';
type CSSVariable = `--${CSSProperty}`;
// '--color' | '--font-size' | '--background'

// CRUD 액션
type Entity = 'user' | 'post' | 'comment';
type Action = 'create' | 'read' | 'update' | 'delete';
type Permission = `${Action}_${Entity}`;

// API 엔드포인트
type Method = 'GET' | 'POST' | 'PUT' | 'DELETE';
type Endpoint = '/users' | '/posts';
type Route = `${Method} ${Endpoint}`;

마치며

TypeScript의 타입 시스템은 강력하지만, 올바르게 사용하는 것이 중요합니다.

핵심 원칙:

  1. any 최소화: unknown, 제네릭, 유틸리티 타입으로 대체
  2. 타입 좁히기: typeof, in, instanceof, 사용자 정의 타입 가드 활용
  3. 재사용성: 제네릭과 유틸리티 타입으로 DRY 원칙 지키기
  4. 점진적 적용: 레거시 코드는 단계적으로 마이그레이션
  5. 명확한 의도: 타입이 복잡할 땐 주석으로 설명

처음에는 타입 정의가 번거롭게 느껴질 수 있지만, 한번 제대로 정의해두면:

  • 자동완성으로 개발 속도 향상
  • 리팩토링이 안전하고 쉬워짐
  • 런타임 에러가 컴파일 타임에 잡힘
  • 코드가 문서화되어 협업이 쉬워짐

any는 정말 어쩔 수 없을 때만 사용하고, 그것도 최소한으로만! 대부분의 경우 unknown, 제네릭, 유틸리티 타입으로 충분히 해결할 수 있습니다.
