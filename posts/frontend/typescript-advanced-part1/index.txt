## 유틸리티 타입

TypeScript가 제공하는 내장 유틸리티 타입들은 기존 타입을 변환하여 새로운 타입을 만듭니다.

### Partial<T>

모든 속성을 optional으로 만듭니다.

```typescript
interface User {
  id: number;
  name: string;
  email: string;
  age: number;
  address: string;
}

// 원하는것 - 모든 필드가 선택적이 됨
type PartialUser = Partial<User>;
// {
//   id?: number;
//   name?: string;
//   email?: string;
//   age?: number;
//   address?: string;
// }

// 일부 필드만 업데이트할 때 유용
const updates: Partial<User> = {
  name: 'Bob',
  age: 30
  // id, email, address는 없어도 OK
};
```

 업데이트 함수

```typescript
// 모든 필드를 받음
function updateUserBad(id: number, updates: User): User {
  const currentUser = getUserById(id);
  return { ...currentUser, ...updates };
  // 문제: id, name 등을 모두 제공해야 함
}

// Partial 사용
function updateUser(id: number, updates: Partial<User>): User {
  const currentUser = getUserById(id);
  return { ...currentUser, ...updates };
}

// 사용
updateUser(1, { name: 'New Name' }); // 이름만 변경
updateUser(1, { age: 26, email: 'new@example.com' }); // 여러 필드 변경
updateUser(1, {}); // 아무것도 변경 안함

// React setState와 유사한 패턴
function setState<T>(state: T, updates: Partial<T>): T {
  return { ...state, ...updates };
}

const user = { id: 1, name: 'Alice', age: 25 };
const newUser = setState(user, { age: 26 }); // { id: 1, name: 'Alice', age: 26 }
```

Partial의 한계 - 한단계에서만 적용

```typescript
interface User {
  id: number;
  name: string;
  profile: {
    bio: string;
    avatar: string;
  };
}

type PartialUser = Partial<User>;
// {
//   id?: number;
//   name?: string;
//   profile?: {  // optional
//     bio: string;      // 여전히 필수
//     avatar: string;   // 여전히 필수
//   };
// }

// 깊은 Partial이 필요하면 직접 구현
type DeepPartial<T> = {
  [P in keyof T]?: T[P] extends object ? DeepPartial<T[P]> : T[P];
};

type DeepPartialUser = DeepPartial<User>;
// {
//   id?: number;
//   name?: string;
//   profile?: {
//     bio?: string;      //optional
//     avatar?: string;   // optional
//   };
// }
```

### Required<T>

반대도 있어야겠죠. 모든 속성을 필수로 만듭니다. 

```typescript
interface Config {
  host?: string;
  port?: number;
  timeout?: number;
  debug?: boolean;
}

// 모든 필드가 필수가 됨
type RequiredConfig = Required<Config>;
// {
//   host: string;
//   port: number;
//   timeout: number;
//   debug: boolean;
// }

// 기본값이 있어야함
const defaultConfig: Required<Config> = {
  host: 'localhost',
  port: 3000,
  timeout: 5000,
  debug: false
};

function createConfig(config: Config = {}): Required<Config> {
  return {
    host: config.host ?? defaultConfig.host,
    port: config.port ?? defaultConfig.port,
    timeout: config.timeout ?? defaultConfig.timeout,
    debug: config.debug ?? defaultConfig.debug
  };
}

// 사용
const config1 = createConfig(); // 모든 필드가 기본값
const config2 = createConfig({ port: 8080 }); // port만 변경, 나머지는 기본값
```

### Readonly<T>

모든 속성을 읽기 전용으로 만듭니다.

```typescript
interface User {
  id: number;
  name: string;
  email: string;
}

type ReadonlyUser = Readonly<User>;
// {
//   readonly id: number;
//   readonly name: string;
//   readonly email: string;
// }

const user: ReadonlyUser = {
  id: 1,
  name: 'Alice',
  email: 'alice@example.com'
};

user.name = 'Bob'; // 에러
```

function reducer(state: ImmutableState, action: Action): ImmutableState {
  // state.count = 10; // 에러

    case 'SET_USER':
      return { ...state, user: action.payload };

  }
}

### Omit<T, K>

특정 속성을 제외합니다. Pick의 반대.

```typescript
interface User {
  id: number;
  name: string;
  email: string;

// password 제외
type PublicUser = Omit<User, 'password'>;
// { id: number; name: string; email: string }

// 여러 필드 제외
type UserWithoutMeta = Omit<User, 'id' | 'createdAt' | 'updatedAt'>;
```

 DTO (Data Transfer Object) 생성

```typescript
interface User {
  id: number;
  name: string;
  email: string;
  password: string;
  createdAt: Date;
  updatedAt: Date;
}

// API 응답에서 민감한 정보 제외
type UserDTO = Omit<User, 'password'>;

function getUserDTO(user: User): UserDTO {
  const { password, ...rest } = user;
  return rest;
}

// 생성 시 자동 생성 필드 제외
type CreateUserDTO = Omit<User, 'id' | 'createdAt' | 'updatedAt'>;

async function createUser(data: CreateUserDTO): Promise<User> {
  const user: User = {
    ...data,
    id: generateId(),
    createdAt: new Date(),
    updatedAt: new Date()
  };

  await saveToDatabase(user);
  return user;
}

// 사용
const newUser = await createUser({
  name: 'Alice',
  email: 'alice@example.com',
  password: 'hashed_password'
  // id, createdAt, updatedAt는 필요 없음
});

// 업데이트용 DTO (id 제외, 나머지 선택적)
type UpdateUserDTO = Partial<Omit<User, 'id' | 'createdAt'>>;

async function updateUser(id: number, data: UpdateUserDTO): Promise<User> {
  const user = await getUserById(id);
  return {
    ...user,
    ...data,
    updatedAt: new Date()
  };
}

// 사용
await updateUser(1, { name: 'New Name' });
await updateUser(1, { email: 'new@email.com', password: 'new_hash' });
```

### Extract<T, U>와 Exclude<T, U>

Union 타입에서 특정 타입을 추출하거나 제외합니다.

```typescript
type T1 = 'a' | 'b' | 'c' | 'd';
type T2 = 'a' | 'c' | 'e';

// Extract: 교집합 
type Common = Extract<T1, T2>; // 'a' | 'c'

// Exclude: 차집합 
type Unique = Exclude<T1, T2>; // 'b' | 'd'

//  null/undefined 제거
type Value = string | number | null | undefined;
type NonNullableValue = Exclude<Value, null | undefined>; // string | number

// TypeScript 내장 NonNullable과 같음
type NonNullable<T> = Exclude<T, null | undefined>;
```

이벤트 타입 필터링

```typescript
// 모든 이벤트 타입
type AllEvents = 'click' | 'scroll' | 'resize' | 'keydown' | 'keyup' | 'mousemove';

// 마우스 이벤트만 추출
type MouseEvent = Extract<AllEvents, 'click' | 'mousemove'>; // 'click' | 'mousemove'

// 마우스 이벤트 제외
type NonMouseEvent = Exclude<AllEvents, MouseEvent>;
// 'scroll' | 'resize' | 'keydown' | 'keyup'

// 함수 시그니처에서 특정 타입만 추출
type Fn1 = () => void;
type Fn2 = (x: number) => void;
type Fn3 = (x: string, y: number) => void;

type AllFn = Fn1 | Fn2 | Fn3;

// void를 반환하는 함수만 추출
type VoidFn = Extract<AllFn, (...args: any[]) => void>;
// Fn1 | Fn2 | Fn3 (모두 void 반환)

// 매개변수가 없는 함수만 추출
type NoArgsFn = Extract<AllFn, () => any>;
// Fn1
```

### ReturnType<T>와 Parameters<T>

함수 타입에서 반환 타입과 매개변수 타입을 추출합니다.

```typescript
// ReturnType: 함수의 반환 타입 추출
function getUser() {
  return { id: 1, name: 'Alice', email: 'alice@example.com' };
}

type User = ReturnType<typeof getUser>;
// { id: number; name: string; email: string }

// async 함수
async function fetchData() {
  return { data: 'hello' };
}

type FetchReturn = ReturnType<typeof fetchData>;
// Promise<{ data: string }>

// Promise를 벗기려면 Awaited 사용 
type FetchData = Awaited<ReturnType<typeof fetchData>>;
// { data: string }

//함수의 매개변수 타입 추출 (튜플)
function createUser(name: string, age: number, email: string) {
  return { id: Date.now(), name, age, email };
}

type CreateUserParams = Parameters<typeof createUser>;
// [name: string, age: number, email: string]

const params: CreateUserParams = ['Alice', 25, 'alice@example.com'];
const user = createUser(...params);
```

함수 래퍼 (Wrapper)

```typescript
// 로깅 추가
function withLogging<T extends (...args: any[]) => any>(fn: T) {
  return (...args: Parameters<T>): ReturnType<T> => {
    console.log(`[${fn.name}] Called with:`, args);
    const result = fn(...args);
    console.log(`[${fn.name}] Returned:`, result);
    return result;
  };
}

// 사용
const add = (a: number, b: number) => a + b;
const loggedAdd = withLogging(add);

loggedAdd(2, 3);
// [add] Called with: [2, 3]
// [add] Returned: 5

function memoize<T extends (...args: any[]) => any>(fn: T) {
  const cache = new Map<string, ReturnType<T>>();

  return (...args: Parameters<T>): ReturnType<T> => {
    const key = JSON.stringify(args);

    if (cache.has(key)) {
      console.log('Cache hit!');
      return cache.get(key)!;
    }

    const result = fn(...args);
    cache.set(key, result);
    return result;
  };
}

// 사용
const expensiveCalculation = (n: number) => {
  console.log('Computing...');
  return n * n;
};

const memoized = memoize(expensiveCalculation);

memoized(5); // "Computing..." >25
memoized(5); // "Cache hit!" >25
```

## any 

`any`는 TypeScript의 타입 시스템을 완전히 우회하는 역할을 합니다. 
하지만 애니를 남발할 경우 TS를 사용하는 의미가 없다고 생각합니다. 

### any가 위험한 이유

```typescript
let data: any = 'hello';

data.toUpperCase(); 
data.toFixed(); // 런타임 에러
data.nonExistentMethod(); // 런타임 에러
data.property.nested.deep.value; // 런타임 에러

// 컴파일에 잡히지 않는 버그
function processUser(user: any) {
  console.log(user.name.toUpperCase()); // user.name이 없으면
  console.log(user.age + 1); // age가 문자열이면
  return user.email.toLowerCase(); // email이 없으면
}

processUser({}); //  런타임 에러
processUser({ name: 123 }); // 런타임 에러
processUser({ name: 'Alice', age: '25' }); // NaN

// 타입 추론 망가뜨림
const result: any = fetchData();
const value = result.data; 
const doubled = value * 2; 
const upper = value.toUpperCase(); // 싹다 any 타입

```

### unknown: 안전한 any

`unknown`은 any와 비슷하지만 타입 체크를 강제합니다.

```typescript

let value: unknown = 'hello';

// 타입 체크 없이는 사용 불가
value.toUpperCase(); // 에러
value.length; // 에러
value(); // 에러

// 타입 체크 후 사용 가능
if (typeof value === 'string') {
  console.log(value.toUpperCase()); // 스트링으로 유추
}

// 여러 타입 체크
function processUnknown(value: unknown) {
  if (typeof value === 'string') {
    return value.toUpperCase();
  }

  if (typeof value === 'number') {
    return value.toFixed(2);
  }

  if (Array.isArray(value)) {
    return value.length;
  }

  return 'Unknown type';
}
```

JSON 파싱

```typescript
// any 사용
function parseJSONBad(json: string): any {
  return JSON.parse(json);
}

const data = parseJSONBad('{"name": "Alice"}');
console.log(data.name); // 타입 안전하지 않음

// unknown + 타입 가드
function safeJsonParse(json: string): unknown {
  try {
    return JSON.parse(json);
  } catch {
    return null;
  }
}

const data = safeJsonParse('{"name": "Alice"}');

// 타입 가드로 안전하게 사용
if (isUser(data)) {
  console.log(data.name); // User 타입
  console.log(data.email);
}

// API 응답
async function fetchAPI(url: string): Promise<unknown> {
  const response = await fetch(url);
  return response.json();
}

const result = await fetchAPI('/api/user');

if (isUser(result)) {
  console.log(result.email);
}
```

### any를 unknown으로 대체하기

```typescript
// any 사용
function processDataBad(data: any) {
  return data.value * 2;
}

// unknown + 타입 가드
function processData(data: unknown): number {
  if (
    typeof data === 'object' &&
    data !== null &&
    'value' in data &&
    typeof (data as { value: unknown }).value === 'number'
  ) {
    return (data as { value: number }).value * 2;
  }

  throw new Error('Invalid data format');
}

// 또는 타입 가드 함수로 분리
interface DataWithValue {
  value: number;
}

function isDataWithValue(data: unknown): data is DataWithValue {
  return (
    typeof data === 'object' &&
    data !== null &&
    'value' in data &&
    typeof (data as DataWithValue).value === 'number'
  );
}

function processDataBetter(data: unknown): number {
  if (isDataWithValue(data)) {
    return data.value * 2; 
  }

  throw new Error('Invalid data format');
}
```

### Type Assertion 최소화

타입 단언(`as`)은 컴파일러를 속이는 것이므로 최소화해야 합니다.

```typescript
const user = JSON.parse(userString) as User;
console.log(user.name); // user가 User 타입이 아니면 에러!

// 검증 후 사용
function parseUser(json: string): User | null {
  try {
    const data = JSON.parse(json);
    if (isUser(data)) {
      return data;
    }
    return null;
  } catch {
    return null;
  }
}

const user = parseUser(userString);
if (user) {
  console.log(user.name); 
}


const element = document.getElementById('myId') as HTMLInputElement;
element.value = 'test'; // element가 input이 아니면..?


const element = document.getElementById('myId');
if (element instanceof HTMLInputElement) {
  element.value = 'test';
}

// 또는 옵셔널 체이닝
const value = (element as HTMLInputElement | null)?.value;
```

### 제네릭으로 대체하기

```typescript

function wrapInArrayBad(value: any): any[] {
  return [value];
}

const nums = wrapInArrayBad(42); // any[]
const first = nums[0]; // any

// 제네릭 사용
function wrapInArray<T>(value: T): T[] {
  return [value];
}

const nums = wrapInArray(42); // number[]
const first = nums[0]; // number

function mapObjectBad(obj: any, fn: any): any {
  const result: any = {};
  for (const key in obj) {
    result[key] = fn(obj[key]);
  }
  return result;
}

function mapObject<T extends Record<string, any>, U>(
  obj: T,
  fn: (value: T[keyof T]) => U
): Record<keyof T, U> {
  const result = {} as Record<keyof T, U>;
  for (const key in obj) {
    result[key] = fn(obj[key]);
  }
  return result;
}

const nums = { a: 1, b: 2, c: 3 };
const doubled = mapObject(nums, x => x * 2);
// Record<"a" | "b" | "c", number>
```

### any가 허용되는 경우

완전히 피할 수 없는 경우도 있습니다. 이럴 경우 주석으로 남기는 방법을 선택합니다.

// 외부 라이브러리 타입이 없을 때 (임시)

 */
// eslint-disable-next-line @typescript-eslint/no-explicit-any
function logAnything(data: any): void {
  console.log(JSON.stringify(data, null, 2));

// 테스트일 경우
const mockFn = jest.fn() as any;
