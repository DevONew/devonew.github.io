AI를 프로젝트에 학습시키기

Claude Code의 진짜 힘은 프로젝트별로 AI를 학습시킬 수 있다는 점이다.

.claude/ 디렉토리: 팀의 AI 지식 베이스

your-project/
├── .claude/
│   ├── agents/           # 서브에이전트
│   │   ├── code-reviewer.md
│   │   └── test-runner.md
│   ├── skills/           # 워크플로우 지식
│   │   ├── coding-style/
│   │   ├── git-workflow/
│   │   └── deployment/
│   └── prompts/          # 재사용 프롬프트
└── ...

이 디렉토리를 Git에 커밋하면:

  • 팀 전체가 같은 AI 경험 공유
  • 신입이 와도 즉시 팀 규칙 학습됨
  • 프로젝트 컨텍스트가 코드와 함께 버전 관리됨

서브에이전트 (Sub-agents)

서브에이전트란?

서브에이전트는 특정 작업을 처리하는 특화된 AI 워커다. 마치 팀에서 각자 전문 분야가 있는 개발자들처럼, 각 서브에이전트는 자신의 역할에 최적화된 지침과 도구를 가진다.

왜 서브에이전트인가?

1. 컨텍스트 보존

각 서브에이전트는 별도 컨텍스트에서 작동합니다:

주 대화: "인증 모듈 리팩토링해줘"
  ↓
서브에이전트 (code-reviewer): 별도 컨텍스트에서 리뷰
  ↓
주 대화: 리뷰 결과만 받음 (세부 과정은 주 컨텍스트 오염 안 함)

2. 특화된 전문성

각 서브에이전트는 특정 도메인에 집중:

  • code-reviewer: 코드 품질, 보안, 베스트 프랙티스
  • debugger: 루트 원인 분석, 에러 해결
  • test-runner: 테스트 실행, 실패 수정

3. 재사용성

한 번 만들면 모든 프로젝트에서 재사용 가능하다.

서브에이전트 생성하기

/agents

대화형 메뉴가 열립니다:

  • 모든 서브에이전트 보기
  • 새 서브에이전트 생성
  • 기존 서브에이전트 편집
  • 도구 권한 관리

권장 방법: Claude에게 먼저 생성 요청

/agents
> code-reviewer 서브에이전트 만들어줘

Claude가 자동으로:
1. 적절한 description 작성
2. 필요한 도구 선택 (Read, Grep, Glob, Bash)
3. 시스템 프롬프트 생성

서브에이전트 파일 구조

서브에이전트는 마크다운 파일로 저장된다:

---
name: code-reviewer
description: 코드 품질, 보안, 유지보수성 리뷰. 코드 작성/수정 직후 자동 사용.
tools: Read, Grep, Glob, Bash
model: sonnet
---

시니어 코드 리뷰어로서 높은 품질 기준을 보장한다.

호출 시:
1. git diff로 최근 변경사항 확인
2. 수정된 파일에 집중
3. 즉시 리뷰 시작

리뷰 우선순위:
- 심각한 문제 (반드시 수정)
- 경고 (수정 권장)
- 제안 (개선 고려)

저장 위치

유형위치범위
프로젝트.claude/agents/현재 프로젝트 (팀 공유)
개인~/.claude/agents/모든 프로젝트 (개인용)

서브에이전트 사용하기

자동 위임

Claude가 자동으로 적절한 서브에이전트 호출:

> 최근 변경사항 코드 리뷰해줘
[자동으로 code-reviewer 서브에이전트 사용]

명시적 호출

특정 서브에이전트를 직접 요청:

> code-reviewer 서브에이전트로 PR 리뷰해줘
> test-runner 서브에이전트로 실패한 테스트 고쳐줘
> debugger 서브에이전트로 이 에러 조사해줘

실전 서브에이전트 예시

디버거

---
name: debugger
description: 에러, 테스트 실패, 예상치 못한 동작 디버깅. 문제 발생 시 자동 사용.
tools: Read, Edit, Bash, Grep, Glob
---

루트 원인 분석 전문 디버거입니다.

호출 시:
1. 에러 메시지와 스택 트레이스 캡처
2. 재현 단계 식별
3. 실패 위치 격리
4. 최소한의 수정 구현
5. 솔루션 검증

각 문제에 대해:
- 루트 원인 설명
- 진단을 뒷받침하는 증거
- 구체적인 코드 수정
- 테스트 접근법
- 재발 방지 권장사항

테스트 러너

---
name: test-runner  
description: 테스트 자동 실행 및 실패 수정. 코드 변경 후 자동 사용.
tools: Bash, Read, Edit, Grep
---

테스트 자동화 전문가입니다.

코드 변경 감지 시:
1. 관련 테스트 자동 실행
2. 실패 시 원인 분석
3. 원래 테스트 의도 유지하며 수정
4. 재실행으로 검증

Agent Skills

Agent Skills란?

Agent Skills는 팀의 워크플로우와 규칙을 AI에게 가르치는 방법입니다. 서브에이전트가 "누구"라면, Skills는 "무엇을 어떻게"를 정의합니다.

Skills가 호출되는 방식

Skills는 자동으로 발견됩니다:

사용자: "새 컴포넌트 만들어줘"
  ↓
Claude: "team-coding-style" Skill 발견
  ↓
Claude: Skill 지침 읽고 자동 적용
  ↓
결과: 팀 스타일 가이드를 따르는 컴포넌트

Skill 생성하기

Personal Skill (개인용)

mkdir -p ~/.claude/skills/my-skill

Project Skill (팀 공유)

mkdir -p .claude/skills/team-coding-style

SKILL.md 작성

---
name: team-coding-style
description: 우리 팀 코딩 스타일. 모든 코드 작성 시 자동 적용.
---

# 팀 코딩 스타일 가이드

## React 컴포넌트

1. **함수형 컴포넌트 사용** (클래스형 X)
2. **Props는 interface로 정의**
3. **파일명은 PascalCase** (Button.tsx)
4. **styled-components 사용** (CSS 파일 X)

```typescript
// ✅ 좋은 예
interface ButtonProps {
  label: string;
  onClick: () => void;
}

export const Button: React.FC<ButtonProps> = ({ label, onClick }) => {
  return <StyledButton onClick={onClick}>{label}</StyledButton>;
};

네이밍

  • 변수/함수: camelCase
  • 상수: UPPER_SNAKE_CASE
  • 타입/인터페이스: PascalCase

API 호출

  • axios 사용 (fetch X)
  • /api 폴더에 집중
  • 에러 처리 필수

### 팀과 Skills 공유

```bash
# 1. Skill 생성
mkdir -p .claude/skills/git-workflow

# 2. SKILL.md 작성
# (커밋 메시지 규칙, 브랜치 전략 등)

# 3. Git 커밋
git add .claude/
git commit -m "Add team Git workflow Skill"
git push

# 4. 팀원이 자동으로 받기
git pull
# 이제 Claude는 팀 Git 규칙 자동 적용

실전 Skill 예시: Git 워크플로우

---
name: git-workflow
description: 팀 Git 커밋/PR 규칙. 커밋 메시지 작성 시 자동 적용.
---

# Git 워크플로우

## 브랜치 전략

- `main`: 프로덕션
- `develop`: 개발
- `feature/기능명`: 기능 개발
- `hotfix/문제명`: 긴급 수정

## 커밋 메시지

type: subject (50자 이내)

body (선택사항)

Refs: #이슈번호


**Type**:
- feat: 새 기능
- fix: 버그 수정
- docs: 문서
- refactor: 리팩토링
- test: 테스트

**예시**:

feat: 사용자 프로필 편집 기능 추가

  • 프로필 이미지 업로드
  • 닉네임/소개 수정
  • 유효성 검사 추가

Refs: #123

이제 Claude는:

  • 커밋 메시지를 자동으로 팀 형식으로 작성
  • 적절한 브랜치 이름 제안
  • PR 템플릿 자동 작성

Skill 도구 권한 제한

보안을 위해 Skill이 사용할 도구를 제한할 수 있습니다:

---
name: safe-file-reader
description: 읽기 전용 파일 접근. 변경 없이 확인만 필요할 때 사용.
allowed-tools: Read, Grep, Glob
---

# Safe File Reader

읽기 전용 파일 접근을 제공합니다.

## Instructions
1. Read로 파일 내용 확인
2. Grep으로 파일 내 검색
3. Glob으로 패턴별 파일 찾기

파일 수정은 불가능합니다.

이 Skill이 활성화되면 Claude는 권한 요청 없이 Read, Grep, Glob만 사용합니다.
