## 왜 Claude Code인가?

ChatGPT, GitHub Copilot, Cursor 등 여러 AI 코딩 도구가 있는데, 왜 Claude Code를 배워야 할까요?

### ChatGPT/Copilot과의 결정적 차이

**1. 실행 권한이 있는 AI**

물론 많은 코드들이 정말 좋지만 ChatGPT는 코드를 "제안"만 하지 복사-붙여넣기 및 적용은 본인이 해야한다.

```
ChatGPT: "이 코드를 사용하세요"
개발자: 복사 → 에디터 열기 → 붙여넣기 → 실행
```

Claude Code는 **직접 파일을 수정하고 실행**한다!!:

```bash
> 인증 버그 고쳐줘

Claude Code:
1. 파일 읽기 (auth.ts)
2. 문제 진단
3. 코드 수정
4. 테스트 실행
5. 결과 확인
→ 모두 자동으로 수행
```

**2. 프로젝트 전체 컨텍스트 이해**

Copilot은 현재 열린 파일만 탐색이 가능하지만 Claude Code는 **전체 코드베이스를 이해**한다:

```bash
> 로그인 기능이 어디서 처리되는지 찾아줘

Claude Code:
- Grep으로 프로젝트 전체 검색
- 관련 파일 찾기 (auth.ts, login.tsx, api.ts)
- 각 파일의 연관 관계 파악
- 데이터 플로우 설명
```

**3. 학습 가능한 AI**

ChatGPT나 gemini는 기존 웹서치를 이용한 방대한 데이터들을 학습하지만 상대적으로 개인 프로젝트 학습이 쉽지않고 오래걸린다. 
그렇지만 그에 비해 Claude Code는 **프로젝트별 학습**이 가능하다.:

```bash
# 팀의 코딩 규칙을 저장
.claude/skills/coding-style/SKILL.md

# 이제 Claude는 항상 팀 규칙을 따름
> 새 컴포넌트 만들어줘
[자동으로 팀 스타일 가이드 적용]
```

**4. 자율적 문제 해결**

Claude Code는 스스로 파일을 탐색하며 어떤 부분에서 에러가 나는지 확인하고 스스로 해결한다.

```bash
> 테스트 실패 원인 찾아줘

Claude Code:
1. 테스트 실행
2. 에러 로그 분석
3. 관련 코드 검사
4. 문제 발견
5. 수정 적용
6. 재실행으로 검증
→ 결과만 보고
```

### 브라우저 vs 터미널: 근본적인 차이

**브라우저 AI (ChatGPT, Claude.ai)**:
- 복사-붙여넣기 워크플로우
- 컨텍스트 제한적
- 매번 새로 설명 필요

**터미널 AI (Claude Code)**:
- 직접 파일 시스템 접근
- 프로젝트 컨텍스트 유지
- 학습된 워크플로우 재사용


## Claude Code 시작하기

### 설치

```bash
npm install -g @anthropic-ai/claude-code
```

### 첫 실행

```bash
claude
```

프롬프트가 나타나면 질문을 입력하세요:

```bash
> 이 프로젝트 구조 설명해줘
```

### 실행 가능한 도구들

Claude Code는 다음 도구를 사용할 수 있다:

- **Read/Write**: 파일 읽기/쓰기
- **Edit**: 코드 수정 (정확한 위치 지정)
- **Bash**: 모든 터미널 명령 실행
- **Grep/Glob**: 프로젝트 전체 검색
- **Git**: 버전 관리 자동화

## 슬래시 커맨드 가이드

특수 커맨드로 작업을 더 효율적으로 수행 할 수 있고 이것은 앞으로 내가 계슉 학습하고 적용해야하는 일이다.

### 필수 커맨드

**`/add` - 파일 컨텍스트 추가**
```bash
/add src/auth.ts

# 특정 파일을 대화 컨텍스트에 추가
# 이후 질문은 이 파일 기준으로 답변
```

**`/drop` - 파일 컨텍스트 제거**
```bash
/drop src/old-code.ts

# 더 이상 필요 없는 파일을 컨텍스트에서 제거
```

**`/clear` - 컨텍스트 초기화**
```bash
/clear

# 대화 기록 지우고 새로 시작
# 토큰 초과 시 유용
```

**`/help` - 도움말**
```bash
/help

# 모든 커맨드와 사용법 표시
```

### 고급 커맨드

**`/architect` - 아키텍처 설계**
```bash
/architect

# 실행 없이 계획만 세움
# 큰 변경 전 구조 설계할 때 유용
```

**`/review` - 코드 리뷰**
```bash
/review

# 최근 변경사항 리뷰
# git diff 기반으로 자동 분석
```

**`/debug` - 디버그 모드**
```bash
/debug

# 상세한 로그 출력
# 문제 해결 시 유용
```

## 실전 프롬프트 예시

### 초보자용: 기본 작업

**파일 탐색**
```bash
> 이 프로젝트 구조 설명해줘
> API 관련 파일들이 어디 있어?
> 테스트 파일 찾아줘
```

**코드 이해**
```bash
> 이 함수가 뭐하는 거야? (특정 함수 보면서)
> 로그인 플로우 전체 설명해줘
> 이 에러는 왜 나는 거야?
```

**간단한 수정**
```bash
> 이 변수명 더 명확하게 바꿔줘
> 주석 추가해줘
> console.log 다 지워줘
```

### 중급자용: 복잡한 작업

**기능 구현**
```bash
> 사용자 프로필 편집 기능 만들어줘. 
  기존 스타일 따라서 하고, 
  유효성 검사도 추가해줘

> 다크모드 지원 추가해줘. 
  localStorage에 설정 저장하고,
  시스템 설정도 따라가게

> 무한 스크롤 구현해줘.
  Intersection Observer 사용하고,
  로딩 상태도 표시
```

**리팩토링**
```bash
> 이 컴포넌트 너무 커. 작은 컴포넌트들로 쪼개줘
> 중복 코드 제거하고 유틸 함수로 만들어줘
> 이 코드 TypeScript로 변환해줘
```

**디버깅**
```bash
> 로그인 후 리다이렉트가 안 돼. 원인 찾아줘
> 메모리 누수 있는지 확인해줘
> 왜 이 테스트가 간헐적으로 실패하는지 분석해줘
```

### 고급자용: 프로젝트 수준 작업

**아키텍처 변경**
```bash
> Redux를 Zustand로 마이그레이션해줘.
  기존 기능 유지하면서 단계적으로 진행

> 모놀리식 API를 MSA로 분리하고 싶어.
  먼저 분리 계획 세워주고, 
  인증 서비스부터 분리해줘
```

**성능 최적화**
```bash
> Lighthouse 점수 90점 이상 만들어줘.
  번들 사이즈 분석부터 시작

> 이 페이지 초기 로딩 2초 이내로 줄여줘.
  Code splitting, lazy loading 다 적용
```

**자동화**
```bash
> PR 전에 자동으로 체크할 GitHub Action 만들어줘.
  린트, 테스트, 빌드 확인하고 커버리지 체크

> 배포 파이프라인 구축해줘.
  dev, staging, prod 환경 분리하고
  자동 롤백 기능도 추가
```

## 실전 워크플로우

### 커맨드 조합: 새 기능 개발

```bash
# 1. 계획 세우기
/architect
> 사용자 알림 기능 구현 계획 세워줘

# 2. 관련 파일 추가
/add src/components/Notification.tsx
/add src/hooks/useNotification.ts

# 3. 구현
> 계획대로 구현해줘

# 4. 리뷰
/review
> 방금 작업 리뷰해줘

# 5. 테스트
> 테스트 작성하고 실행해줘
```

### 커맨드 조합: 버그 수정

```bash
# 1. 디버그 모드 활성화
/debug

# 2. 문제 재현
> 로그인 버그 재현해줘

# 3. 원인 분석
> 로그 보고 원인 찾아줘

# 4. 수정
> 원인 해결하고 테스트해줘

# 5. 정리
/clear
```
