## 프로젝트 소개

Voice AI App은 다국어 언어 학습을 위한 음성 기반 AI 튜터 애플리케이션입니다. 사용자가 음성으로 질문하면 AI가 원어민 선생님 역할로 답변하며, TTS(Text-to-Speech)로 자연스러운 음성으로 응답합니다.

### 어쩌다 만들게 되었나

친구가 언어 학습을 위한 나만의 원어민 AI 선생님이 필요하다고 했습니다. 시중에는 여러 AI 음성 서비스가 있지만, 새로운 서비스에 추가로 결제하기보다는 이미 결제해둔 AI 서비스(Claude와 ElevenLabs)를 활용하고 싶었습니다.
게다가 이번 기회에 Claude Code도 학습해보고 싶었습니다. 프로젝트 초기 설정부터 컴포넌트 구조, 상태 관리까지 Claude Code와 함께 작업하며 AI 페어 프로그래밍 경험을 쌓을 수 있었습니다.


## 전체 동작 플로우

사용자가 앱을 사용하는 전체 과정은 다음과 같습니다:

```
1. 사용자가 마이크 버튼 클릭
    ↓
2. 음성 녹음 시작 (마이크 권한 요청)
    ↓
3. 실시간 음량 분석 (버튼 크기 동적 변화)
    ↓
4. 침묵 감지 (1.5초) → 자동 녹음 종료
    ↓
5. [STT] ElevenLabs API 호출
    ↓
6. 음성 → 텍스트 변환
    ↓
7. 변환된 텍스트 2초간 화면 표시
    ↓
8. [CHAT] Claude API 호출
    ↓
9. AI 응답 생성 (대화 히스토리 포함)
    ↓
10. 언어 자동 감지 (한국어/영어/일본어/중국어)
    ↓
11. [TTS] ElevenLabs API 호출 (또는 캐시에서 가져오기)
    ↓
12. 텍스트 → 음성 변환
    ↓
13. 음성 재생 + 텍스트 표시
    ↓
14. 재생 완료 → 초기 상태로 복귀
```

## 핵심 구현: 상태 관리

이 프로젝트에서 가장 중요한 부분은 **상태 관리**입니다. 앱은 5가지 상태를 가지며, 각 상태마다 UI와 동작이 완전히 다릅니다.

### 상태 타입 정의

```typescript
type AppState = 'idle' | 'listening' | 'processing' | 'speaking' | 'error';

interface UseAppStateReturn {
  appState: AppState
  displayText: string              // 사용자가 말한 내용 (STT 결과)
  responseText: string             // AI 응답 텍스트
  errorMessage: string
  conversationHistory: ConversationHistory  // 대화 히스토리
  audioBlob: Blob | null          // TTS 오디오
  isAudioPlaying: boolean
  currentLanguage: string         // 현재 언어 (kr, fr)
  setAppState: (state: AppState) => void
  setDisplayText: (text: string) => void
  setResponseText: (text: string) => void
  // ... 기타 setter 함수들
  getStatusText: (appState: AppState, displayText: string, responseText: string) => string
}
```

### 상태 전환 플로우

```
idle (대기 중)
  ↓ [마이크 버튼 클릭]
listening (듣는 중)
  ↓ [침묵 감지 1.5초 → STT 완료]
listening (STT 결과 표시 2초)
  ↓ [2초 후 자동 전환]
processing (생각하는 중)
  ↓ [Chat API 응답 완료]
speaking (말하는 중)
  ↓ [TTS 재생 완료]
idle (초기 상태 복귀)
```

### useAppState 훅 - 상태 관리 개선

초기에는 `switch` 문으로 상태별 텍스트를 관리했지만, 실무에서는 객체 매핑 패턴을 더 많이 사용한다는 것을 알게 되었습니다. `switch` 문은 상태가 추가될 때마다 여러 줄의 코드를 수정해야 하고, TypeScript의 타입 체크를 완전히 활용하지 못하는 단점이 있습니다. 객체 매핑 패턴으로 리팩토링하면 `Record<AppState, ...>` 타입으로 모든 상태를 컴파일 타임에 보장할 수 있고, 데이터와 로직이 한곳에 모여 있어 유지보수가 훨씬 쉬워집니다.

```typescript
// 개선 후: 객체 매핑 패턴
const getStatusText = useCallback((state: AppState, displayTxt: string, responseTxt: string): string => {
  const statusTextMap: Record<AppState, string | ((displayTxt: string, responseTxt: string) => string)> = {
    idle: '탭하여 시작',
    listening: (displayTxt) => displayTxt || '듣는 중...',
    processing: '생각하는 중...',
    speaking: (_, responseTxt) => responseTxt,
    error: '탭하여 시작',
  }

  const status = statusTextMap[state]
  return typeof status === 'function' ? status(displayTxt, responseTxt) : status
}, [])
```

이 패턴은 Redux, Zustand 같은 상태 관리 라이브러리에서도 널리 사용되며, 함수형 프로그래밍 패러다임과 잘 맞아 React 생태계에서 선호됩니다.

### StateViews 컴포넌트 - 상태별 화면 렌더링

각 상태별로 완전히 다른 UI를 렌더링하는 컴포넌트입니다. Framer Motion의 `AnimatePresence`를 활용해 상태 전환 시 부드러운 애니메이션을 구현했습니다.

```typescript
export function StateViews({ appState, transcript, responseText, volumeLevel, ... }: StateViewsProps) {
  return (
    <AnimatePresence mode="wait">
      {/* listening 상태: 실시간 음량에 따라 버튼 크기 변화 */}
      {appState === 'listening' && (
        <div key="listening">
          <p>{transcript || '듣는중'}</p>
          <VoiceButton
            scale={Math.min(0.8 + (volumeLevel / 100) * 0.3, 1.1)}
            isListening={true}
          />
        </div>
      )}

      {/* speaking 상태: 응답 텍스트 + 작은 버튼 */}
      {appState === 'speaking' && responseText && (
        <motion.div key="speaking" initial={{ opacity: 0 }} animate={{ opacity: 1 }}>
          <ResponseDisplay text={responseText} />
          <VoiceButton size={80} />
        </motion.div>
      )}

      {/* idle, processing, error 상태 생략 */}
    </AnimatePresence>
  )
}
```

`AnimatePresence`의 `mode="wait"` 옵션으로 이전 상태가 완전히 사라진 후 다음 상태가 나타나도록 했습니다. `listening` 상태에서는 실시간 `volumeLevel`에 따라 버튼 크기가 동적으로 변화하며, 각 상태별로 고유한 `key`를 부여해 React가 정확히 애니메이션을 처리하도록 구현했습니다.

## 상태별 UI 구현

각 상태마다 완전히 다른 UI를 표시합니다. 프로젝트에서 구현한 각 상태의 특징을 살펴보겠습니다.

### 1. idle 상태 - 초기 화면

![메인 페이지](/image/main-page.png)

화면 중앙에 부드러운 그라데이션 원형 버튼이 위치하며, 그 아래 "탭하여 시작"이라는 안내 문구가 표시됩니다. 미니멀한 디자인으로 사용자가 직관적으로 버튼을 탭하도록 유도합니다.

**디자인 문제와 해결**

초기 디자인에서 몇 가지 크로스 브라우징 이슈가 발생했습니다:

**문제 1: 박스 섀도우 번짐 현상**
- 컴퓨터 화면에서는 사각형 보더가 보이지 않는데, 아이폰에서는 보더가 선명하게 보이는 문제
- 확인해보니 `box-shadow` 값이 과도하게 설정되어 있었음

```tsx
// 개선 전: 과도한 box-shadow
className="shadow-[0_0_40px_rgba(0,0,0,0.3)]"

// 개선 후: 부드러운 그림자
className="shadow-lg"  // Tailwind의 기본값 사용
```

**문제 2: iOS Safari 하단 바 겹침**
- 아이폰 16 Pro에서 하단 네비게이션 바가 버튼을 가리는 문제
- `padding-bottom`을 아무리 넣어도 해결되지 않음
- `env(safe-area-inset-bottom)`을 활용하여 해결 (iOS의 홈 인디케이터 높이만큼 자동으로 패딩 추가)

```tsx
// 개선 후
<div className="pb-[calc(2rem+env(safe-area-inset-bottom))]">
  <VoiceButton />
</div>
```

```tsx
<motion.button
  onClick={handleButtonClick}
  whileHover={{ scale: 1.05 }}
  whileTap={{ scale: 0.95 }}
  className="w-32 h-32 rounded-full bg-gradient-to-br from-gray-800 to-black shadow-lg"
>
  탭하여 시작
</motion.button>
```
