## 프로젝트 소개

**Voice AI App**은 다국어 언어 학습을 위한 음성 기반 AI 튜터 애플리케이션입니다. 사용자가 음성으로 질문하면 AI가 원어민 선생님 역할로 답변하며, TTS(Text-to-Speech)로 자연스러운 음성으로 응답합니다.

여행 중 "이탈리아 여행 가서 반드시 알아야 할 문장 딱 3개만 소개해드릴게요"라고 말하면, AI가 실시간으로 이를 인식하고 음성으로 답변해주는 것을 상상해보세요. 바로 이 프로젝트가 실현하는 경험입니다.

![메인 페이지](/image/main-page.png)

### 핵심 기능

- 🎤 **음성 인식 (STT)**: ElevenLabs를 통한 다국어 음성 인식
- 🤖 **AI 대화**: Claude Sonnet 4 기반 원어민 튜터 역할
- 🔊 **음성 재생 (TTS)**: ElevenLabs 자연스러운 음성 합성
- 🌏 **다국어 지원**: 한국어, 영어, 일본어, 프랑스어 등 동시 인식
- 💾 **캐싱 시스템**: TTS 응답 재사용으로 비용 절감 및 성능 향상

## 왜 이 프로젝트를 만들었나?

친구가 언어 학습을 위한 나만의 원어민 AI 선생님 음성 AI가 필요하다고 했습니다. 시중에는 여러 AI 음성 서비스가 있지만, 새로운 서비스에 추가로 결제하기보다는 **이미 결제해둔 AI 서비스(Claude와 ElevenLabs)를 활용**하고 싶었습니다.

### 기존 언어 학습 앱의 한계

기존 언어 학습 도구들을 사용하면서 여러 불편함을 느꼈습니다:

- ❌ **단일 언어 인식**: 한 번에 한 언어만 인식 가능
  - 한국어 모드에서는 영어를 말해도 인식 못함
  - 매번 언어를 수동으로 전환해야 함
- ❌ **텍스트 기반 대화**: 음성 대화가 부자연스러움
- ❌ **비싼 비용**: 전문 튜터는 시간당 수만 원
- ❌ **접근성**: 시간과 장소의 제약

### 해결책

이미 구독 중인 Claude API와 ElevenLabs API를 조합하여 다음과 같은 문제들을 해결했습니다:

- ✅ **다국어 동시 인식**: 한국어로 질문하고 영어로 대답받기 가능
- ✅ **자연스러운 대화**: 음성 기반 양방향 소통
- ✅ **24/7 이용**: 언제 어디서나 학습 가능
- ✅ **추가 비용 없음**: 기존 구독 서비스만으로 무제한 대화

## 기술 스택

### Frontend
- **Framework**: Next.js 14 (App Router)
- **Language**: TypeScript
- **UI**: React 18 + Tailwind CSS
- **Animation**: Framer Motion

### Backend & APIs
- **Claude API**: Anthropic Claude Sonnet 4 (claude-sonnet-4-20250514)
- **ElevenLabs STT**: Scribe v2 (음성 → 텍스트)
- **ElevenLabs TTS**: Multilingual v2 (텍스트 → 음성)

### Infrastructure
- **Hosting**: Vercel
- **Repository**: GitHub

**기술 스택 선택 이유**:

**Next.js 14 App Router**를 선택한 이유는 서버/클라이언트 컴포넌트를 명확하게 분리할 수 있고, API Routes로 백엔드 로직을 간단하게 구현할 수 있기 때문입니다. 특히 API 키를 서버에서만 관리해 보안을 강화할 수 있었습니다.

**Claude Sonnet 4**는 다국어 이해 능력이 뛰어나고, Prompt Caching을 통해 비용을 80% 이상 절감할 수 있습니다. 원어민 튜터 역할을 수행하기에 충분한 성능을 제공합니다.

**ElevenLabs**는 29개 이상 언어를 자동 감지하고, 자연스러운 음성 합성을 제공합니다. Web Speech API와 달리 다국어를 동시에 인식할 수 있어 언어 학습 앱에 최적입니다.

## 전체 동작 플로우

사용자가 앱을 사용하는 전체 과정은 다음과 같습니다:

```
1. 사용자가 마이크 버튼 클릭
    ↓
2. 음성 녹음 시작 (마이크 권한 요청)
    ↓
3. 실시간 음량 분석 (버튼 크기 동적 변화)
    ↓
4. 침묵 감지 (1.5초) → 자동 녹음 종료
    ↓
5. [STT] ElevenLabs API 호출
    ↓
6. 음성 → 텍스트 변환
    ↓
7. 변환된 텍스트 2초간 화면 표시
    ↓
8. [CHAT] Claude API 호출
    ↓
9. AI 응답 생성 (대화 히스토리 포함)
    ↓
10. 언어 자동 감지 (한국어/영어/일본어/중국어)
    ↓
11. [TTS] ElevenLabs API 호출 (또는 캐시에서 가져오기)
    ↓
12. 텍스트 → 음성 변환
    ↓
13. 음성 재생 + 텍스트 표시
    ↓
14. 재생 완료 → 초기 상태로 복귀
```

## 핵심 구현: 상태 관리

이 프로젝트에서 가장 중요한 부분은 **상태 관리**입니다. 앱은 5가지 상태를 가지며, 각 상태마다 UI와 동작이 완전히 다릅니다.

### 상태 머신 (State Machine)

```typescript
// 5가지 상태 정의
type AppState = 'idle' | 'listening' | 'processing' | 'speaking' | 'error';
```

상태 전환 플로우:

```
idle (대기 중)
  ↓ [마이크 버튼 클릭]
listening (듣는 중)
  ↓ [침묵 감지 1.5초 → STT 완료]
listening (STT 결과 표시 2초)
  ↓ [2초 후 자동 전환]
processing (생각하는 중)
  ↓ [Chat API 응답 완료]
speaking (말하는 중)
  ↓ [TTS 재생 완료]
idle (초기 상태 복귀)
```

### useAppState 훅 구현

상태를 관리하는 커스텀 훅을 개발했습니다:

```typescript
const [appState, setAppState] = useState<AppState>('idle');

const handleButtonClick = useCallback(async () => {
  if (appState === 'idle') {
    setAppState('listening');
    await startRecording();
  } else if (appState === 'listening') {
    await stopRecording();
  } else if (appState === 'speaking') {
    setAppState('listening');
    await startRecording();
  }
}, [appState]);
```

**핵심 포인트**:
- 버튼 클릭 시 **현재 상태에 따라** 다음 상태로 전환
- `useCallback`으로 불필요한 리렌더링 방지
- 각 상태는 명확한 진입/종료 조건을 가짐

## 상태별 UI 구현

각 상태마다 완전히 다른 UI를 표시합니다. 프로젝트에서 구현한 각 상태의 특징을 살펴보겠습니다.

### 1. idle 상태 - 초기 화면

![초기 화면](/image/main-page.png)

**특징**:
- 중앙에 검은 원형 버튼
- "탭하여 시작" 안내 문구
- 미니멀한 디자인


Framer Motion을 활용해 버튼 hover 시 부드러운 스케일 애니메이션을 적용했습니다.

### 2. listening 상태 - 음성 인식 중

![듣는 중](/image/listening.png)

**특징**:
- "듣는중" 텍스트 표시
- 음량에 따라 버튼 크기 동적 변화
- 실시간 음량 분석

```typescript
const { transcript, volumeLevel, startRecording, stopRecording } = 
  useVoiceRecorderStreaming(
    setAppState, 
    undefined, 
    handleFinalTranscript, 
    currentLanguage, 
    onError
  );
```

**Web Audio API로 실시간 음량 계산**:

```typescript
// FFT 분석으로 음량 계산
const analyser = audioContext.createAnalyser();
analyser.fftSize = 256;
const dataArray = new Uint8Array(analyser.frequencyBinCount);

// 실시간 음량 추출
const getVolumeLevel = () => {
  analyser.getByteFrequencyData(dataArray);
  const sum = dataArray.reduce((a, b) => a + b, 0);
  const average = sum / dataArray.length;
  return Math.min(100, average * 1.5);  // 0-100 범위로 정규화
};
```

음량 레벨에 따라 버튼 크기가 동적으로 변하는 **펄스 효과**를 구현했습니다:

```tsx
<motion.div
  animate={{
    scale: 1 + (volumeLevel / 100) * 0.5  // 음량이 클수록 버튼 확대
  }}
  transition={{ duration: 0.1 }}
>
  {/* 버튼 */}
</motion.div>
```

### 3. 침묵 감지 기능

사용자가 말을 멈추면 자동으로 녹음을 종료하는 기능입니다.

```typescript
const SILENCE_THRESHOLD = 15;    // 음량 임계값
const SILENCE_DURATION = 1500;   // 침묵 시간 (1.5초)

if (volumeLevel < SILENCE_THRESHOLD && 
    silenceDuration >= SILENCE_DURATION) {
  stopRecording();  // 자동 종료
}
```

**동작 원리**:
1. 음량이 15 미만이면 "침묵"으로 간주
2. 1.5초 동안 침묵이 지속되면 녹음 자동 종료
3. ElevenLabs Scribe v2 API 호출해 음성 → 텍스트 변환

### 4. STT 결과 표시 (2초)

![듣는 중 최대화](/image/listening-max.png)

STT 완료 후 사용자가 자신이 말한 내용을 확인할 수 있도록 **2초 동안 화면에 표시**합니다.

```typescript
// STT 완료 → 2초 표시 → processing 전환
setTimeout(() => {
  setAppState('processing');
}, 2000);
```

**왜 2초인가?**
- ElevenLabs STT는 실시간 중간 결과를 제공하지 않음 (HTTP 방식)
- 최종 결과만 받기 때문에 사용자 확인 시간 필요
- 2초는 텍스트를 읽기에 적당한 시간
