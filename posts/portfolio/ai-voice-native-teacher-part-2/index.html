Part 1에서는 프로젝트 개요와 상태 관리를 다뤘습니다. Part 2에서는 listening, processing, speaking을 살펴보겠습니다.

listening 상태 - 음성 인식 중

사용자가 마이크 버튼을 탭하면 음성 녹음이 시작되고, 실시간으로 음량을 분석하여 시각적 피드백을 제공합니다.

listening 상태에서는 세 가지 핵심 기술이 동작합니다.

  • Web Audio API의 AnalyserNode로 실시간 음량을 분석하여 버튼 크기를 동적으로 변경합니다.
  • 2초간 침묵이 지속되면 자동으로 녹음을 중지하여 사용자가 매번 버튼을 다시 누를 필요가 없도록 했습니다.
  • 마지막으로 ElevenLabs STT를 사용하여 한국어, 영어, 일본어 등 다국어를 동시에 인식할 수 있습니다.

문제: Web Speech API의 한계

초기에는 브라우저 내장 Web Speech API를 사용했으나, 한 번에 하나의 언어만 인식 가능한 문제가 있었습니다. 사용자가 외국어로 질문하면 STT를 한국어(ko-KR)로 설정했을 때 영어 발음을 한글로 잘못 인식했고, 영어(en-US)로 바꾸면 이번엔 한국어가 제대로 인식되지 않았습니다. 사용자가 어떤 언어로 말할지 예측할 수 없기 때문에, 다국어 동시 인식이 가능한 ElevenLabs Scribe v2로 전환했습니다.

ElevenLabs STT 구현

ElevenLabs Scribe v2는 language_codes 파라미터에 여러 언어를 동시에 지정할 수 있습니다. 예를 들어 한국어를 기본으로 하되, 사용자가 현재 학습 중인 언어(프랑스어, 일본어 등)를 추가로 설정하면 두 언어를 동시에 인식합니다.

// app/api/stt/route.ts
export async function POST(request: Request) {
  const sttFormData = new FormData();
  sttFormData.append('file', audioFile);
  sttFormData.append('model_id', 'scribe_v2');

  // 다국어 동시 인식 (한국어 + 현재 학습 중인 언어)
  sttFormData.append('language_codes', 'ko');
  if (currentLanguage && currentLanguage !== 'ko') {
    sttFormData.append('language_codes', currentLanguage);
  }

  const response = await fetch('https://api.elevenlabs.io/v1/speech-to-text', {
    method: 'POST',
    headers: { 'xi-api-key': process.env.ELEVENLABS_API_KEY },
    body: sttFormData,
  });

  return Response.json({ text: result.text });
}

사용자가 "How do you say thank you in French?"라고 영어로 말하든, "프랑스어로 감사합니다 어떻게 말해?"라고 한국어로 말하든 정확하게 인식됩니다.

실시간 음량 분석 및 시각화

사용자가 말하는 동안 버튼 크기를 실시간으로 변화시켜 시각적 피드백을 제공합니다. Web Audio API의 AnalyserNode를 사용하여 주파수 데이터를 추출하고, 이를 0-100 범위로 정규화한 후 버튼의 scale 값으로 변환합니다.

// app/hooks/useVoiceRecorderStreaming.ts
const updateVolume = useCallback(() => {
  const dataArray = new Uint8Array(analyserRef.current.frequencyBinCount);
  analyserRef.current.getByteFrequencyData(dataArray);

  // 평균 음량 계산 (0-255 → 0-100)
  const average = dataArray.reduce((a, b) => a + b) / dataArray.length;
  const normalizedVolume = Math.min(100, (average / 255) * 100);

  setVolumeLevel(normalizedVolume);
  animationFrameRef.current = requestAnimationFrame(updateVolume);
}, []);

계산된 음량 값을 버튼 컴포넌트에 전달하면, 음량이 높을수록 버튼이 커지고(최대 1.1배), 낮을수록 작아집니다(최소 0.8배).

// app/components/VoiceButton.tsx
<VoiceButton
  scale={Math.min(0.8 + (volumeLevel / 100) * 0.3, 1.1)}
  isListening={true}
/>

침묵 감지 자동 종료

사용자 경험을 개선하기 위해 침묵 감지 기능을 추가했습니다. 사용자가 말을 마치고 2초 동안 침묵이 지속되면 자동으로 녹음을 중지하여, 매번 버튼을 다시 눌러야 하는 번거로움을 제거했습니다.

// app/hooks/useVoiceRecorderStreaming.ts
const setupSilenceDetection = useCallback((mediaRecorder: MediaRecorder) => {
  const checkSilence = () => {
    if (lastVolumeRef.current < AUDIO_CONFIG.SILENCE_THRESHOLD) {
      mediaRecorder.stop();  // 침묵 감지 → 자동 중지
    } else {
      // 음성 있으면 타이머 리셋
      silenceTimeoutRef.current = setTimeout(checkSilence, AUDIO_CONFIG.SILENCE_DURATION);
    }
  };

  silenceTimeoutRef.current = setTimeout(checkSilence, AUDIO_CONFIG.SILENCE_DURATION);
}, []);

침묵 판단 기준은 음량 15 이하(0-100 스케일)이며, 이 상태가 2000ms(2초) 지속되면 녹음을 종료합니다.

// app/constants/audio.ts
export const AUDIO_CONFIG = {
  SILENCE_THRESHOLD: 15,    // 침묵 판단 음량 (0-100 중 15 이하)
  SILENCE_DURATION: 2000,   // 2초간 침묵 지속 시 자동 종료
  FFT_SIZE: 256,
};

STT 결과 2초간 표시

녹음이 종료된 후 STT 결과를 즉시 Chat API로 보내지 않고, 2초간 화면에 표시하여 사용자가 자신이 말한 내용을 확인할 수 있도록 했습니다. 이는 사용자가 음성 인식이 제대로 되었는지 시각적으로 확인할 수 있어 UX를 크게 개선합니다.

// STT 완료 시
setTranscript(recognizedText);

// 2초 동안 listening 상태 유지 (사용자가 확인)
setTimeout(() => {
  setAppState('processing');  // 이제 Chat API 호출
  onFinalTranscript(recognizedText);
}, 2000);

processing 상태 - AI 응답 생성

STT로 변환된 텍스트를 Claude AI에게 보내 응답을 생성합니다. 사용자는 "생각하는 중.." 메시지를 보며 대기하지만, 내부적으로는 Chat API 호출과 언어 감지가 진행됩니다.

Claude API 호출

STT로 변환된 텍스트는 Anthropic의 Claude API로 전송됩니다. 현재 대화 히스토리와 새로운 사용자 메시지를 함께 보내면, Claude가 컨텍스트를 이해하고 적절한 응답을 생성합니다. 시스템 프롬프트는 언어 선생님 역할을 하도록 설정되어 있으며, Prompt Caching을 사용하여 API 비용을 절감합니다(자세한 내용은 Part 3에서 다룹니다).

// app/api/chat/route.ts
export async function POST(request: Request) {
  const { message, conversationHistory = [] } = await request.json();

  const response = await anthropic.messages.create({
    model: 'claude-sonnet-4-20250514',
    max_tokens: 200,
    system: [
      {
        type: 'text',
        text: SYSTEM_PROMPT,
        cache_control: { type: 'ephemeral' },  // 프롬프트 캐싱 (Part 3에서 자세히)
      },
    ],
    messages: [...conversationHistory, { role: 'user', content: message }],
  });

  return Response.json({ response: response.content[0].text });
}

useChatHandler 훅 - Chat과 TTS 오케스트레이션

useChatHandler는 STT 결과를 받아 Chat API 호출, 언어 감지, TTS 처리를 순차적으로 실행하는 오케스트레이션 역할을 합니다. 먼저 사용자 입력에서 언어를 감지하여 STT 설정을 업데이트하고, Chat API로 응답을 받은 후 즉시 TTS API를 호출하여 음성을 생성합니다. 모든 준비가 완료되면 speaking 상태로 전환하여 사용자에게 음성을 재생합니다.

// app/hooks/useChatHandler.ts
const handleFinalTranscript = useCallback((finalText: string) => {
  // 1. 사용자 입력에서 언어 감지
  const detectedLanguage = detectLanguage(finalText);
  if (detectedLanguage) onLanguageDetected(detectedLanguage);

  // 2. Chat API 호출 → 3. TTS 처리 → 4. speaking 상태 전환
  handleChatAPI(finalText, conversationHistory)
    .then(async (aiResponse) => {
      onResponseReceived(aiResponse);
      const audioBlob = await handleTTSAPI(aiResponse);

      setTimeout(() => {
        onStateChange('speaking');
        onAudioGenerated(audioBlob);
      }, 200);
    })
    .catch(onError);
}, [conversationHistory, handleChatAPI, handleTTSAPI]);

언어 자동 감지

사용자가 "프랑스어로 말해줘" 또는 특정 언어를 언급하면, 해당 언어를 자동으로 감지하여 STT/TTS 설정을 업데이트합니다. 간단한 키워드 매칭 방식을 사용하며, 한국어와 영어 모두 지원합니다.

// app/utils/language-detector.ts
export const LANGUAGE_MAP: { [key: string]: string } = {
  '프랑스': 'fr', '프랑스어': 'fr', 'france': 'fr',
  '일본': 'ja', '일본어': 'ja', 'japan': 'ja',
  '스페인': 'es', '이탈리아': 'it',
};

export function detectLanguage(text: string): string | undefined {
  const lowerText = text.toLowerCase();

  for (const [key, code] of Object.entries(LANGUAGE_MAP)) {
    if (lowerText.includes(key)) return code;
  }

  return undefined;
}

예를 들어 사용자가 "이탈리아 여행 문장 알려줘"라고 말하면 it 코드가 감지되고, 이후 Claude가 "Buongiorno! Come stai?"라고 답변하면 STT가 이탈리아어로 설정되어 정확한 발음 인식이 가능해집니다.

speaking 상태 - TTS 및 오디오 재생

AI 응답이 생성되면 ElevenLabs TTS API를 호출해 음성으로 변환하고 재생합니다.

ElevenLabs TTS API 호출

Claude의 응답 텍스트를 ElevenLabs TTS API로 전송하여 자연스러운 음성으로 변환합니다. eleven_multilingual_v2 모델을 사용하여 다국어를 지원하며, Safari 브라우저와의 호환성을 위해 mp3_44100_128 포맷과 Accept-Ranges: bytes 헤더를 설정했습니다.

// app/api/tts/route.ts
export async function POST(request: Request) {
  const { text, voiceId = 'XB0fDUnXU5powFXDhCwa' } = await request.json();

  const response = await fetch(
    `https://api.elevenlabs.io/v1/text-to-speech/${voiceId}`,
    {
      method: 'POST',
      headers: {
        'xi-api-key': process.env.ELEVENLABS_API_KEY,
        'Content-Type': 'application/json',
      },
      body: JSON.stringify({
        text,
        model_id: 'eleven_multilingual_v2',  // 다국어 지원
        output_format: 'mp3_44100_128',  // Safari 호환성
      }),
    }
  );

  const audioBuffer = await response.arrayBuffer();

  return new Response(audioBuffer, {
    headers: {
      'Content-Type': 'audio/mpeg',
      'Accept-Ranges': 'bytes',  // Safari 필수 헤더
    },
  });
}

문제: iOS Safari에서 소리가 안 나오는 이슈

데스크톱 Chrome과 Firefox에서는 정상 작동했지만, iOS Safari에서만 음성이 재생되지 않는 문제가 발생했습니다. iOS는 보안과 데이터 절약을 위해 사용자의 명시적인 인터랙션 없이 자동으로 오디오를 재생하는 것을 제한합니다. 또한 <audio> 태그에 playsInline 속성이 없으면 전체화면 비디오 플레이어로 전환되거나 재생이 실패합니다. 마지막으로 load() 메서드를 명시적으로 호출하지 않으면 iOS Safari에서 오디오 소스가 제대로 로드되지 않습니다.

이러한 문제를 해결하기 위해 세 가지 조치를 취했습니다. 먼저 <audio> 태그에 playsInline 속성을 추가하여 iOS에서 전체화면 모드로 전환되지 않도록 했습니다.

// app/components/AudioPlayer.tsx
<audio
  ref={audioRef}
  playsInline        // iOS 전체화면 방지 (필수!)
  preload="auto"
  onEnded={handleAudioEnd}
/>

다음으로 오디오 소스를 설정한 후 load() 메서드를 명시적으로 호출하여 iOS Safari에서 오디오 파일이 제대로 로드되도록 했습니다.

// app/components/AudioPlayer.tsx
useEffect(() => {
  if (isPlaying && audioBlob) {
    const audioUrl = URL.createObjectURL(audioBlob);
    audioRef.current.src = audioUrl;

    audioRef.current.load();  // iOS Safari 필수
    audioRef.current.play().catch(console.error);
  }
}, [isPlaying, audioBlob]);

마지막으로 플랫폼 감지 유틸리티를 만들어 iOS 기기와 Safari 브라우저를 정확하게 식별할 수 있도록 했습니다. iOS 13+ iPad는 데스크톱 User-Agent를 사용하므로 ontouchend 이벤트 존재 여부로 추가 확인합니다.

// app/utils/platform-detect.ts
export const isIOS = (): boolean => {
  const ua = window.navigator.userAgent;
  const isIOSDevice = /iPad|iPhone|iPod/.test(ua);
  const isIPadOS = ua.includes('Mac') && 'ontouchend' in document;

  return isIOSDevice || isIPadOS;
};

export const isIOSSafari = (): boolean => {
  return isIOS() && /Safari/.test(ua) && !/Chrome/.test(ua);
};

이제 iOS Safari를 포함한 모든 주요 브라우저에서 음성이 정상적으로 재생됩니다.
