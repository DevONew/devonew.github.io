Part 1에서는 프로젝트 배경, 기술 스택, 그리고 지역 데이터 매칭과 색상 계산 등 기본적인 구현을 다뤘습니다. Part 2에서는 사용자 인터랙션을 구현하고, 전체 데이터 플로우를 완성하는 과정을 소개합니다.

## 핵심 구현 4: 호버 이벤트 처리

폴리곤 호버 시 이벤트 핸들러로 지역 ID를 저장하고, CSS로 테두리 강조 및 밝기 증가 효과를 적용했습니다.

```tsx
<polygon
  points={points}
  fill={regionColor}
  onMouseEnter={() => onRegionHover?.(region.id)}
  onMouseLeave={() => onRegionHover?.(null)}
  className="hover:stroke-[#d4c4a8] hover:stroke-[3] hover:brightness-110 
             transition-all duration-200 cursor-pointer"
/>
```

**구현 포인트**:
- `onMouseEnter`: 마우스가 지역 위로 올라갈 때 해당 지역 ID 전달
- `onMouseLeave`: 마우스가 떠날 때 `null`로 초기화
- Tailwind CSS의 `hover:` 유틸리티로 시각적 피드백 제공
  - `stroke-[#d4c4a8]`: 베이지색 테두리
  - `stroke-[3]`: 두께 3px
  - `brightness-110`: 10% 밝게
  - `transition-all`: 부드러운 전환 효과
  - `cursor-pointer`: 마우스 커서 변경

![호버 인터랙션과 폴리곤 강조 효과](/image/hover and polygon.png)

이렇게 하면 사용자가 지도 위에 마우스를 올렸을 때 즉각적인 시각적 피드백을 받을 수 있습니다.

## 핵심 구현 5: 동적 데이터 표시

메인 페이지(`page.tsx`)에서 호버 상태를 관리하고, 우측 패널에 동적으로 데이터를 표시합니다.

```tsx
const [hoveredRegionId, setHoveredRegionId] = useState<string | null>(null);

const hoveredRegionData = hoveredRegionId
  ? data.find((item) => {
      const fullName = item.sdName + item.cityName;
      return fullName.includes(hoveredRegionId);
    })
  : null;

const displayData = hoveredRegionData || 
  data.find((item) => 
    item.sdName === '서울특별시' && item.cityName === '중구'
  ) || 
  data[0];
```

**데이터 선택 로직**:
1. **호버된 지역이 있으면**: 해당 지역 데이터 표시
2. **호버된 지역이 없으면**: 서울 중구 데이터를 기본값으로 표시
3. **서울 중구 데이터도 없으면**: 전국 데이터(0번 인덱스) 표시

이러한 폴백(fallback) 처리로 항상 의미 있는 데이터가 표시되도록 보장합니다.

## 핵심 구현 6: 후보자 카드 동적 렌더링

선거 데이터에서 후보자 메타데이터(이미지, 정당 색상)를 매핑해 후보자 카드를 동적으로 렌더링했습니다.

```tsx
const topCandidates = [
  {
    name: displayData.hbj01,
    vote: displayData.vote01,
    count: displayData.dugsu01,
    party: displayData.jd01,
    meta: candidateMeta[displayData.hbj01] || candidateMeta["이재명"]
  },
  {
    name: displayData.hbj02,
    vote: displayData.vote02,
    count: displayData.dugsu02,
    party: displayData.jd02,
    meta: candidateMeta[displayData.hbj02] || candidateMeta["김문수"]
  },
  {
    name: displayData.hbj03,
    vote: displayData.vote03,
    count: displayData.dugsu03,
    party: displayData.jd03,
    meta: candidateMeta[displayData.hbj03] || candidateMeta["이준석"]
  },
].sort((a, b) => b.vote - a.vote);
```

**구현 세부사항**:
1. 각 후보자의 득표 데이터를 배열로 구성
2. `candidateMeta`에서 이미지와 정당 색상을 가져옴
3. 메타데이터가 없을 경우 기본값 사용
4. 득표율 내림차순으로 정렬
5. 정렬된 순서대로 1위, 2위, 3위 카드 표시

이렇게 하면 지역이 바뀔 때마다 해당 지역의 득표 순위가 자동으로 재계산되어 표시됩니다.

## 전체 데이터 플로우

프로젝트의 전체 데이터 흐름은 다음과 같습니다:

```
1. MSWProvider
   ↓
   Service Worker 초기화
   ↓
2. useElectionData Hook
   ↓
   /api/election 호출
   ↓
   MSW가 요청을 가로채서 mock-data 반환
   ↓
3. page.tsx
   ↓
   데이터 수신 및 상태 관리 (useState)
   ↓
4. 사용자가 지도에 마우스 호버
   ↓
   hoveredRegionId 상태 변경
   ↓
5. React 리렌더링
   ↓
   - CatogramRegion: 호버된 지역 시각적 강조
   - CandidateCard: 호버된 지역의 후보자 정보 표시
   - RegionTurnout: 호버된 지역의 투표율 표시
   ↓
6. 실시간 업데이트 완료
```

### Mock Service Worker (MSW) 설정

개발 환경에서 실제 API 없이 테스트하기 위해 MSW를 사용했습니다.

```typescript
// mocks/handler.ts
import { http, HttpResponse } from 'msw';
import { mockElectionData } from './mock-data';

export const handlers = [
  http.get('/api/election', () => {
    return HttpResponse.json(mockElectionData);
  }),
];
```

```typescript
// app/components/MSWProvider.tsx
'use client';

import { useEffect, useState } from 'react';

export function MSWProvider({ children }: { children: React.ReactNode }) {
  const [mswReady, setMswReady] = useState(false);

  useEffect(() => {
    const initMSW = async () => {
      if (typeof window !== 'undefined') {
        const { worker } = await import('@/mocks/browser');
        await worker.start({
          onUnhandledRequest: 'bypass',
        });
        setMswReady(true);
      }
    };

    initMSW();
  }, []);

  if (!mswReady) {
    return <div>Loading...</div>;
  }

  return <>{children}</>;
}
```

MSW를 사용하면:
- 백엔드 API 없이 프론트엔드 개발 가능
- 실제 API와 동일한 인터페이스로 테스트
- 네트워크 요청을 브라우저 레벨에서 가로채기
- 3,981줄의 상세한 선거 데이터를 즉시 사용 가능

### 데이터 페칭 Hook

```typescript
// hook/useElectionData.ts
import { useState, useEffect } from 'react';
import { ElectionData } from '@/types';

export function useElectionData() {
  const [data, setData] = useState<ElectionData[]>([]);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState<Error | null>(null);

  useEffect(() => {
    const fetchData = async () => {
      try {
        const response = await fetch('/api/election');
        if (!response.ok) throw new Error('Failed to fetch');
        const result = await response.json();
        setData(result);
      } catch (err) {
        setError(err instanceof Error ? err : new Error('Unknown error'));
      } finally {
        setLoading(false);
      }
    };

    fetchData();
  }, []);

  return { data, loading, error };
}
```

이 Hook은:
- 컴포넌트 마운트 시 한 번만 데이터 페칭
- 로딩, 에러 상태 관리
- 타입 안정성 보장

## 마주한 문제와 해결

### 문제 1: 지역명 매칭 불일치
**문제**: 폴리곤의 지역 ID와 선거 데이터의 지역명이 정확히 일치하지 않는 경우가 많았습니다. 예를 들어 "중구"라는 ID가 서울, 부산, 대구에 모두 존재합니다.

**해결**: 
```typescript
function findRegionData(regionId: string, electionData: ElectionData[]) {
  return electionData.find((data) => {
    const fullName = data.sdName + data.cityName;
    // 양방향 부분 문자열 매칭
    return fullName.includes(regionId) || regionId.includes(data.cityName);
  });
}
```

시도명과 시군구명을 합쳐서 비교하고, 양방향 부분 문자열 매칭으로 유연성을 확보했습니다.

### 문제 2: SVG 텍스트 줄바꿈
**문제**: SVG `<text>` 요소는 HTML과 달리 자동 줄바꿈을 지원하지 않습니다.

**해결**:
```tsx
{labelLines.map((line, i) => (
  <tspan
    key={i}
    x={centroid.centerX}
    dy={i === 0 ? `${dyOffset}em` : "1.2em"}
    textAnchor="middle"
  >
    {line}
  </tspan>
))}
```

`<tspan>` 요소를 사용해 수동으로 줄을 나누고, `dy` 속성으로 줄 간격을 조정했습니다.

### 문제 3: 호버 시 깜빡임
**문제**: 폴리곤 경계에서 마우스가 움직일 때 `onMouseEnter`와 `onMouseLeave`가 반복적으로 발생해 깜빡이는 현상이 있었습니다.

**해결**:
```css
pointer-events: bounding-box;
```

SVG의 `pointer-events` 속성을 조정하고, 이벤트 버블링을 적절히 제어해 해결했습니다.

### 문제 4: Mock 데이터 크기
**문제**: 3,981줄의 데이터를 한 번에 로드하면 초기 로딩이 느려질 수 있습니다.

**해결**:
- MSW의 Service Worker를 사용해 백그라운드에서 로딩
- 첫 로딩 시 전국 데이터(0번 인덱스)만 먼저 표시
- 나머지 데이터는 비동기로 로드하면서 사용자는 즉시 인터랙션 가능

1. **Vue.js에서 React로의 마이그레이션**: 두 프레임워크의 차이점과 각각의 장단점을 깊이 이해할 수 있었습니다.

4. **타입 안정성**: TypeScript로 대규모 데이터 구조를 안전하게 관리하는 방법.

### 아쉬운 점
- 초기에 좌표 데이터 정리에 많은 시간 소요
- 250개 폴리곤 각각의 라벨 위치를 수동 조정해야 했던 부분
- 모바일 반응형이 아직 완벽하지 않음

### 개선하고 싶은 점
- 지역 계층 구조를 더 명확하게 표현
- 애니메이션과 전환 효과 추가
- 접근성(Accessibility) 개선
- 성능 최적화 (특히 모바일)

## 마치며

이 프로젝트는 단순한 데이터 시각화를 넘어, 사용자 인터랙션, 상태 관리, 성능 최적화 등 프론트엔드 개발의 여러 측면을 종합적으로 경험할 수 있는 좋은 기회였습니다.

특히 Vue.js로 작성된 기존 코드를 React로 마이그레이션하면서, 단순히 문법을 바꾸는 것이 아니라 React의 철학에 맞게 컴포넌트를 재설계하고 상태 관리 방식을 개선하는 과정에서 많은 것을 배울 수 있었습니다.
